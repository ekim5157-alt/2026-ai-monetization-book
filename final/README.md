# Final

전체 작업의 다섯 번째 단계입니다.

## 프로세스 보호 원칙

- 사용자가 지시하지 않은 Git 연결·생성·저장·우회 방법을 연구하거나 실행하지 않습니다.
- 프로세스상의 문제가 있을 경우, 프로세스 자체를 재정의하거나 하부 프로세스를 만들지 말고 사용자에게 현재 상태를 설명합니다.

같은 모델의 business, research, draft, review 파일을 모두 읽고 최종 승인 원고를 작성합니다.

긴 handoff 문서를 단순 요약하거나 평탄화하지 않습니다. 사례와 세부사항을 압축할 때 실제 성패를 가른 인과관계·임계점·반례·운영 판단을 남깁니다. 핵심 인사이트는 `관찰 → 숨은 변수 → 작동 메커니즘 → 실행 판단 → 실패·전환 신호`로 재구성합니다.

완성본은 `final/model-XX.md`에 저장하며 이 파일만 승인본으로 취급합니다. Sanity 패키지는 final 승인 후 별도 단계에서 생성합니다.
