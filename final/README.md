# Final

전체 작업의 다섯 번째 단계입니다.

같은 모델의 다음 파일을 모두 읽고 최종 승인 원고를 작성합니다.

- `handoff/business/model-XX.md`
- `handoff/research/model-XX.md`
- `handoff/draft/model-XX.md`
- `handoff/review/model-XX.md`

최종 편집 원칙:
- 검증 지적을 반영하고 중복과 모순을 제거합니다.
- 긴 handoff 문서를 단순 요약하거나 문장을 평균적으로 평탄화하지 않습니다.
- 사례와 세부사항을 압축할 때 실제 성패를 가른 인과관계·임계점·반례·운영 판단을 남깁니다.
- 핵심 인사이트를 `관찰 → 숨은 변수 → 작동 메커니즘 → 실행 판단 → 실패·전환 신호`로 재구성합니다.
- 흔한 조언으로 대체하지 않고 독자가 측정 기준, 우선순위, 제작 상한선, 중단 조건을 정할 수 있게 씁니다.
- 원자료에 없는 원칙을 만들어내지 않으며 반복 확인된 구조와 편집 판단을 구분합니다.
- 가장 선명한 대표 사례·수치·반례는 근거로 남깁니다.
- 문장의 길이를 줄이는 것보다 독자의 의사결정을 바꾸는 현실적인 인사이트를 보존하는 것이 우선입니다.

완성본은 `final/model-XX.md`에 저장하며 이 파일만 승인본으로 취급합니다.

Sanity 패키지는 final 승인 후 별도 단계에서 생성합니다.
