## 프로세스 보호 원칙
- business의 모델사례와 사용자 확정 시장 관찰은 검증·교정·복사하지 않는다.
- 공식 사실, 계약·계정 화면 확인 필요, 사용자 제공 운영 관찰, 편집 판단을 분리한다.

## 핵심 결론
소형 AI SaaS·GPT·웹앱의 공식 정책상 병목은 `반복되는 좁은 문제 × 10분 안의 첫 가치 × 사용자당 AI·서버·결제·지원 원가 × 잔존·해지·결제 실패 × 개인정보 보존·삭제와 장애 대응`입니다. 가입자 수보다 첫 결과 완료·재사용·유료 전환·30일 잔존·사용자당 공헌이익을 봐야 합니다.

## 성패를 가르는 것
- `누가 어떤 자료를 넣고 어떤 결과를 몇 번 다시 쓰는가`를 한 문장으로 정의합니다.
- 회원가입 전후의 입력 단계를 줄이고 예시 데이터로 첫 결과를 빠르게 보여줍니다.
- 기능별 평균·최대 API·저장·전송·지원비를 측정합니다.
- 사용자·조직별 무료 한도와 긴 입력·파일·이미지·음성·재생성 상한을 둡니다.
- 결제 요청·인증·승인을 서버에서 검증하고 결제 상태와 서비스 권한을 대조합니다.
- 해지·결제 실패·재시도·환불·데이터 삭제를 제품 기능으로 설계합니다.

## 핵심 인사이트
- 구체적 관찰: 가입·첫 결과·활성·유료·잔존·수납·순이익은 서로 다른 사건입니다.
- 남들이 놓치는 숨은 변수: 사용자당 최대 비용, 무료 악용, 결제 실패, 지원시간, 데이터 삭제, 외부 API 장애입니다.
- 결과가 달라지는 작동 메커니즘: 반복 문제를 좁히고 입력·출력·비용·오류·권한을 통제하면 사용자가 늘어도 마진을 관리할 수 있습니다.
- 임계점·시간차·교환비용: 유입→첫 결과→반복 사용→결제→해지·환불·결제 실패→PG 정산→입금의 시간차가 있습니다.
- 초보자가 먼저 측정하거나 결정할 것: 문제 빈도·첫 가치 시간·사용자당 비용 상한·결제 방식·지원 범위입니다.
- 사업을 계속할 조건: 7일·30일 재사용과 비용 차감 이익이 함께 개선되는 경우입니다.
- 중단·전환 신호: 가입만 늘고 재사용이 없거나 활성 사용자가 늘수록 손실이 커지는 경우입니다.
- 대표 사례·수치·반례: 월 1회 쓰는 범용 챗봇보다 매주 쓰는 견적·보고·분류 도구가 더 높은 결제 가치를 만들 수 있습니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: API·결제·데이터 보존 조건은 공식 사실이며 첫 가치 10분과 사용자 약 20명 검증은 편집 판단입니다.

## 시장 현실
- 시장조사를 통해 먼저 확인한 동향: business의 자료 요약·회의 정리·음성·영상 제작·목적별 문서·문서 기반 Q&A·업종별 결과 생성·마이크로 도구 유형을 유지합니다.
- 반복된 패턴: 빈 대화창보다 URL·파일·대본·녹음 같은 좁은 입력과 다운로드 가능한 결과를 앞에 둡니다.
- 상충되는 의견: 월 구독과 건별 결제는 사용 빈도·원가 변동·지원 구조에 따라 적합성이 다릅니다.
- 작성자 이해관계 또는 광고 가능성: 개발·호스팅·노코드 도구 판매자는 출시 속도와 가입자를 강조할 유인이 있습니다.
- 이미 포화된 부분: 범용 글쓰기·일반 요약·프롬프트만 감싼 챗봇·차별 없는 이미지 생성입니다.
- 아직 남은 기회: 한국 업무 서식·특정 업종 데이터·승인 가능한 출력·반복 기록·기존 프로그램 연결이 있는 좁은 도구입니다.

## 판매·수익·광고 경로
- 실제 구매자·독자·시청자·광고주: 시간을 반복적으로 절약하는 개인 사용자·팀 관리자·기업 구매부서입니다.
- 발견·유입 경로: 검색·SNS 시연·콘텐츠·소개·기존 고객입니다.
- 판매 또는 광고가 일어나는 플랫폼: 자체 웹앱·판매페이지·앱마켓·기업 계약입니다.
- 결제수단: 국내 카드·계좌·앱마켓 인앱결제·기업 계좌이체·세금계산서를 따릅니다.
- 수익 지급자: PG사·앱마켓·기업 고객입니다.
- 플랫폼 수수료·광고비·환불·CS 비용: AI API·서버·DB·저장·전송·PG·앱마켓·환불·광고·지원·개발 비용입니다.
- 확정 수익 조건: 결제 승인 뒤 환불·부분취소·결제 실패·해지·차지백 조건을 통과한 금액입니다.
- 실제 입금 시점과 경로: PG·앱마켓·기업 계약의 정산주기 뒤 입금됩니다.
- 반복 가능한 구조인지: 사용자당 마진·잔존·지원시간이 안정되어야 합니다.
- 해외 타깃과 구분할 부분: 한글 사용자·원화 결제·원화 정산을 기준으로 합니다.

## 공식 확인 사실
- OpenAI API는 ChatGPT 구독과 별도 사용량 과금입니다.
- 2026년 6월 22일 표준 처리 기준 GPT-5.5는 입력 100만 토큰당 5달러, 캐시 입력 0.50달러, 출력 30달러입니다.
- GPT-5.4 mini는 입력 0.75달러, 캐시 입력 0.075달러, 출력 4.50달러입니다.
- 웹 검색은 1,000회당 10달러이며 Batch API는 입력·출력 비용을 50% 줄이고 최대 24시간 비동기 처리합니다.
- OpenAI API 데이터는 명시적 공유 동의 없이는 모델 학습에 사용되지 않습니다.
- 기본 악용 모니터링 로그는 대부분의 기능에서 최대 30일 보존될 수 있습니다.
- Responses API는 기본적으로 응답 상태를 최소 30일 저장할 수 있고 일부 상태형 객체는 삭제할 때까지 보존될 수 있습니다.
- 토스페이먼츠 자동결제는 카드와 계좌이체를 지원하며 리스크 검토와 추가 계약이 필요합니다.
- 구독 운영자가 결제주기 스케줄링과 자동결제 승인 API 호출을 직접 구현해야 합니다.
- 구매자가 해지하면 다음 결제를 요청하지 않고 사용하지 않는 빌링키를 삭제할 수 있습니다.
- 토스페이·카카오페이·네이버페이 같은 국내 간편결제는 자동결제 방식에서 지원되지 않습니다.
- 온라인 결제는 요청·인증·승인이 분리되며 승인 전에 주문번호와 금액을 서버에서 검증해야 합니다.
- NAVER Cloud Platform Object Storage는 저장량·API 요청·네트워크 전송을 기준으로 종량 과금합니다.

## 계약·계정 화면 확인 필요
- PG 계약 수수료·정산주기·보증보험·자동결제 심사·환불·차지백 조건.
- 앱마켓 수수료·구독 해지·환불·세금 조건.
- 개인정보 수집근거·국외 이전·보유기간·탈퇴·백업 삭제·관리자 접근기록.
- 서버·DB·스토리지·이메일·분석도구의 원화 요금과 장애·백업 조건.
- 외부 AI·음성·이미지·문서 처리 서비스의 데이터·사용권·종료 조건.

## 사용자 제공 운영 관찰
- business의 첫 결과 완료율·7일 재사용·무료 한도 도달·결제 성공·30일 잔존·사용자당 비용 관찰을 유지합니다.
- 공개 서비스의 회원 수와 홍보 수치를 유료 사용자·매출·잔존으로 환산하지 않습니다.

## 추정 또는 편집 판단
- 첫 가치 10분·인터뷰 10~20건·수동 서비스 5~10건·첫 결제 5~20건·30~90일 잔존은 초기 검증 기준입니다.
- 특정 무료 한도가 유료 전환을 보장하지 않습니다.
- 가입자 수가 많으면 제품·시장 적합성이 입증된다는 주장도 성립하지 않습니다.

## 필요한 제품
OpenAI API — ChatGPT 구독과 별도 사용량 과금. 모델·검색·파일·데이터 보존 조건 확인. 사용자·조직별 비용 상한 필요.

CLOVA Studio — 한글 업무용 모델·도구 후보. 정확한 모델별 원화 요금은 최신 계정 화면 확인.

토스페이먼츠 — 일반결제·자동결제·취소 API 제공. 자동결제는 추가 계약과 직접 스케줄링이 필요.

NAVER Cloud Platform Object Storage — 저장량·요청·전송 종량 과금. 백업·권한·삭제 정책 포함.

## 오늘·3일·7일·30일
- 오늘: 반복 문제 1개·첫 결과·사용자당 비용 상한·결제 방식을 정합니다.
- 3일: 최소 화면과 기능별 비용·오류·지원 기록표를 만듭니다.
- 7일: 사용자 5~10명의 첫 결과와 재사용·비용을 관찰합니다.
- 30일: 약 20명의 활성·유료·잔존·해지·결제 실패·지원시간·순이익을 비교합니다.

## 위험과 주의사항
- API 학습 제외를 데이터 무보존으로 설명하지 않습니다.
- 무제한 무료 사용과 고비용 파일·이미지·음성 기능을 열어두지 않습니다.
- 결제 인증 성공만 보고 서비스를 제공하지 않고 서버에서 승인과 금액을 검증합니다.
- 해지·환불·부분취소·데이터 삭제 절차를 명확히 제공합니다.
- 민감정보를 필요 이상 수집·저장하지 않습니다.
- 장애·외부 API 변경·서비스 종료 때 데이터 내보내기와 공지 절차를 둡니다.

## 총괄이 반드시 확인할 것
- 가입·첫 결과·활성·유료·잔존·수납·순이익을 구분할 것.
- 사용자당 AI·서버·PG·지원비를 포함할 것.
- 결제 요청·인증·승인·해지·환불·정산을 분리할 것.
- OpenAI 가격에는 2026년 6월 22일 기준과 변동 가능성을 붙일 것.
- 학습 제외와 악용 모니터링·상태형 데이터 보존을 구분할 것.
- 무료 한도와 정기결제 구현을 비용·보안 기능으로 다룰 것.
