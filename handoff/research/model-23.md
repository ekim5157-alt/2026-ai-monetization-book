## 프로세스 보호 원칙
- business의 모델사례와 사용자 확정 시장 관찰은 검증·교정·축약하지 않는다.
- 공식 사실, 계약·계정 화면 확인 필요, 사용자 제공 운영 관찰, 편집 판단을 분리한다.
- 개인 모델의 매출 사례보다 복제 뒤 단위이익·품질·창업자 개입시간을 검증한다.

## 핵심 결론
이 모델의 성패는 `이미 검증된 개인 수익모델 × 문서화된 판단·예외 × 역할별 권한 × 독립 손익단위 × 본사 지원시간이 느리게 증가하는 구조 × 고객·브랜드·데이터·수익배분 계약`에서 갈립니다.

교육생·파트너·지점 수가 늘어도 창업자가 모든 견적·검수·클레임·정산을 승인하면 사업은 확장되지 않습니다. 첫 운영자 한 명이 창업자 없이 한 번의 판매·납품·CS·환불·정산 주기를 완료한 뒤에만 다음 단위를 복제해야 합니다.

## 성패를 가르는 것
- 최근 고객 10~30건의 매출보다 환불·재작업·광고·CS·본사 지원시간을 포함한 중앙 단위이익을 먼저 계산합니다.
- 업무를 유입·상담·견적·계약·입력·생산·검수·승인·결제·정산·CS·재구매로 나눕니다.
- 각 단계에 담당자·입력·출력·완료 조건·금지 기준·예외·에스컬레이션 기준을 붙입니다.
- 파트너는 특정 지역·상품군·고객군의 매출과 비용을 함께 책임지는 독립 손익단위로 운영합니다.
- 본사는 브랜드·공급·공통 도구·데이터 구조·품질 기준·정산 체계를 제공합니다.
- 읽기·작성·승인·송금·삭제·관리자 권한을 역할별로 나누고 퇴사·계약 종료 때 회수합니다.
- 수익배분은 총매출보다 환불·광고·공급·인건비·공통비를 반영한 확정 공헌이익을 기준으로 검토합니다.

## 핵심 인사이트
- 구체적 관찰: 공급·셀러 플랫폼은 상품·판매자·교육·전송도구를 함께 운영하고, 판매 통합 솔루션은 API·교육·커스텀서비스를 함께 제공합니다.
- 남들이 놓치는 숨은 변수: 창업자의 말로만 남아 있는 예외 판단, 파트너별 고객·계정·데이터 접근 범위, 파트너 이탈 뒤 고객·자료·미수금 인계입니다.
- 결과가 달라지는 작동 메커니즘: 판단 기준과 예외를 문서화하고 AI가 반복 입력·분류·초안·보고를 맡으면 운영자는 지역·상품·고객의 손익과 품질에 집중할 수 있습니다.
- 임계점·시간차·교환비용: 교육 완료 뒤에도 첫 고객·첫 정산·첫 환불·첫 장애를 혼자 처리할 때까지 본사 지원비가 발생합니다.
- 초보자가 먼저 측정하거나 결정할 것: 파트너당 공헌이익, 본사 개입시간, 오류·환불률, 독립 운영까지 걸린 기간, 계약 종료 인계비용입니다.
- 사업을 계속할 조건: 새 단위가 정해진 기간 안에 품질과 공헌이익을 만들고 추가 단위가 늘어도 본사 지원시간·오류·분쟁이 더 느리게 증가해야 합니다.
- 중단·전환 신호: 교육생만 늘고 실제 운영자가 없거나 창업자 승인 없이는 영업·가격·클레임·정산이 진행되지 않는 경우입니다.
- 대표 사례·수치·반례: 파트너 매출 합계가 커져도 본사 교육·검수·분쟁·도구·미수금 비용이 더 빠르게 늘면 스케일업이 아닙니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: 플랫폼의 공급·판매·교육·광고·다지점 기능은 공식 확인 사실이며 독립 손익단위와 본사 개입시간 기준은 편집 판단입니다.

## 시장 현실
- 시장조사를 통해 먼저 확인한 동향: 도매매는 교육센터·상품공급사센터와 스마트스토어·쿠팡·11번가·롯데ON 전송·주문관리 기능을 함께 운영합니다.
- 반복된 패턴: 온채널은 공급사 등록·판매사 등록·셀러대학을 연결하고, 플레이오토는 통합 판매관리·API 연동·커스텀서비스·정기교육·고객사례를 함께 제공합니다.
- 상충되는 의견: 파트너 수를 빠르게 늘리는 방식과 첫 운영자의 독립 정산 주기를 끝까지 검증하는 방식은 성장 속도와 실패비용이 다릅니다.
- 작성자 이해관계 또는 광고 가능성: 플랫폼·교육사업자는 가입자·상품 수·매출 사례를 강조할 수 있으므로 독립 손익과 이탈·지원비를 별도로 봐야 합니다.
- 이미 포화된 부분: 수강료를 받고 같은 방식으로 돈을 벌 수 있다고 설명하는 교육, 실제 고객·상품·리드가 없는 파트너 모집, 창업자의 개인 코칭에 의존하는 조직입니다.
- 아직 남은 기회: 개인이 이미 검증한 좁은 서비스·판매모델을 업종별 SOP와 자동화로 바꾸고 실제 고객·공급·데이터·도구와 함께 독립 운영자에게 넘기는 구조입니다.

## 판매·수익·광고 경로
- 실제 구매자·독자·시청자·광고주: 최종 상품·서비스 고객, 지역 사업자, 브랜드·공급업체, 기업 고객입니다.
- 발견·유입 경로: 기존 고객 소개, 검색·콘텐츠, 광고, 도매·판매 플랫폼, 지역 커뮤니티, 파트너 직접 영업입니다.
- 판매 또는 광고가 일어나는 플랫폼: 당근비즈니스·스마트스토어·쿠팡·오픈마켓·자체 판매페이지·직접 계약입니다.
- 결제수단: 플랫폼 결제·카드·간편결제·계좌이체·세금계산서 조건을 따릅니다.
- 수익 지급자: 최종 고객·판매 플랫폼·광고주·본사 또는 파트너입니다.
- 플랫폼 수수료·광고비·환불·CS 비용: 공급·제작 원가, 광고비, 결제·플랫폼 비용, 환불·재작업, 운영자 배분, 도구·서버, 교육·검수·분쟁·본사 지원시간입니다.
- 확정 수익 조건: 최종 고객의 구매확정·환불기간·프로젝트 승인 뒤 비용을 차감하고 계약한 배분 기준을 적용한 금액입니다.
- 실제 입금 시점과 경로: 플랫폼 정산일·고객 잔금일·본사와 파트너 정산일이 다르므로 지급 순서와 보류 조건을 계약합니다.
- 반복 가능한 구조인지: 사업단위별 손익·품질·고객·지원시간을 분리해 비교할 수 있고 독립 운영자가 정상·예외 주기를 완료해야 합니다.
- 해외 타깃과 구분할 부분: 국내 고객·원화 결제·원화 정산·국내 플랫폼을 기본으로 봅니다.

## 공식 확인 사실
- 당근비즈니스는 소상공인용 광고와 전문 마케터용 광고 운영·교육 경로를 함께 제공합니다.
- 당근 브랜드프로필은 브랜드 본사가 전국 지점 정보와 프로모션을 등록·관리하는 기능으로 안내됩니다.
- 당근비즈니스는 비즈프로필·견적·예약·상품판매와 광고를 함께 제공합니다.
- 도매매는 교육센터·상품공급사센터·스피드고전송기를 운영하고 스마트스토어·쿠팡·11번가·롯데ON의 상품 전송·주문관리 기능을 제공합니다.
- 온채널은 공급사 등록·판매사 등록·셀러대학을 함께 제공합니다.
- 플레이오토는 통합 판매관리와 API 연동·개발자센터·커스텀서비스·정기교육·고객사례를 함께 제공합니다.
- 플랫폼의 가입자·상품 수·매출 성공사례는 개별 운영자의 독립 손익과 본사의 확장성을 보장하지 않습니다.
- 네이버웍스 Free는 최대 30명과 공용 5GB를 제공하며 관리·보안·감사 기능이 제한됩니다.
- 네이버웍스 월간 계약 기준 Lite는 1인당 월 4,000원, Standard는 8,500원, Standard Plus는 13,000원이며 세금은 별도입니다.
- 네이버웍스는 구성원 추가일부터 일할 과금하며 사용하지 않거나 첫 로그인 전이어도 과금될 수 있습니다.
- 카카오워크 Free는 최대 30명까지 0원이고 월간 기준 Mini 2,900원, Standard 7,900원, Premium 11,900원이며 부가가치세는 별도입니다.
- OpenAI 비즈니스 제품과 API Platform의 비즈니스 데이터는 기본적으로 모델 학습에 사용되지 않습니다.
- 사용자와 OpenAI 사이에서는 사용자가 입력과 적법하게 받은 출력을 소유하지만 조직 내부 접근·보존·삭제·외부 연결 권한은 관리자 정책과 제품 조건을 확인해야 합니다.
- OpenAI API는 ChatGPT 구독과 별도 과금입니다.
- 2026년 6월 22일 표준 처리 기준 GPT-5.5는 입력 100만 토큰당 5달러, 캐시 입력 0.50달러, 출력 30달러입니다.
- GPT-5.4 mini는 입력 0.75달러, 캐시 입력 0.075달러, 출력 4.50달러입니다.
- 브랜드 사용, 일정한 영업방식 준수, 지속적인 교육·지원·통제, 대가 지급이 결합되는 구조는 가맹사업 해당 여부를 법률 검토해야 합니다.
- 공동사업·판매대행·도급·가맹·라이선스에 모든 사업자에게 공통 적용되는 표준 수익배분율은 공식 확인 불가입니다.

## 현실 기준 교정
- 사업자등록 여부: 본사와 독립 운영자의 계약·수금·세금계산서·고객 책임 구조에 따라 각각 사업자등록과 세무 처리가 필요할 수 있습니다. 공동사업 여부는 계약 명칭보다 실제 운영과 수익·비용 부담으로 확인합니다.
- 결제·지급·정산 조건: 최종 고객 결제, 플랫폼 정산, 공급대금, 운영자 배분, 본사 사용료를 주문번호·프로젝트번호 단위로 대조합니다.
- 가격·세금·수수료: 본사 수수료율보다 공헌이익 계산식과 비용 차감 순서·부가가치세·원천징수·세금계산서 책임을 먼저 정합니다.
- 미국 기준 자료인지: 본 실행안은 국내 플랫폼·국내 법률·원화 정산을 기준으로 합니다.

## 계약·계정 화면 확인 필요
- 협력 형태가 판매대행·도급·공동사업·상표 라이선스·가맹사업 중 어디에 해당하는지.
- 고객 계약 주체, 환불·하자·재작업·미수금·보증 책임 주체.
- 매출 기준인지 공헌이익 기준인지, 광고·환불·세금·공급비·공통비 차감 순서.
- 고객·브랜드·콘텐츠·데이터·SOP·자동화·코드·계정의 소유·접근·사용·반환·삭제 범위.
- 파트너별 지역·상품군·고객군·가격 정책과 내부 경쟁 조정 방식.
- 플랫폼·광고·결제·도메인·클라우드 계정의 명의와 관리자·승인·삭제 권한.
- 파트너 이탈·퇴사·계약 종료 때 고객 인계·미수금·환불·자료 회수·접근권한 회수 절차.
- 운영자와 팀원의 근로자성·도급·위임 관계, 업무지시·근무시간·보수·책임 구조.

## 사용자 제공 운영 관찰
- 개인 모델 검증→업무 분해→표준 운영절차→AI 자동화→교육·권한→독립 운영→복수 사업단위 구조를 유지합니다.
- 초기 자본이 생긴 뒤 자체 브랜드 상품 판매대행을 OEM·ODM 화장품·건강기능식품으로 확장하는 방향은 별도 인허가·표시·품질책임 검토 대상으로 남깁니다.
- 당근 커머스의 농산물 박스 판매와 당근 커뮤니티 코칭의 교육 플랫폼화는 앞 모델에서 검증된 뒤 복수 사업단위로 연결합니다.

## 추정 또는 편집 판단
- 최근 고객 10~30건 중앙값, 첫 운영자 한 명, 고객 3건 이내 시험, 30일 독립 정산 주기는 내부 운영 기준입니다.
- 새 사업단위가 2~3개월 안에 손익을 만든다는 기준은 업종별 검증이 필요한 편집 판단입니다.
- 파트너 수·지점 수·교육생 수·자동화 처리율이 늘면 본사 순이익이 자동으로 증가한다는 보장은 없습니다.
- 한 운영자의 성공을 평균 파트너 성과로 일반화하지 않습니다.

## 오늘·3일·7일·30일
- 오늘: 본인이 실제로 돈을 받은 모델 하나의 최근 10~30건 단위이익과 창업자 개입 지점을 계산합니다.
- 3일: 전체 업무를 단계별 담당·입력·출력·완료·금지·예외·에스컬레이션 기준으로 문서화합니다.
- 7일: 다른 사람 한 명에게 고객 1~3건을 맡기고 질문·오류·본사 개입시간·권한 문제를 기록합니다.
- 30일: 운영자 한 명이 창업자 없이 판매·납품·CS·환불·정산 주기를 완료하게 하고 단위 공헌이익·품질·지원시간으로 복제 여부를 결정합니다.

## 위험과 주의사항
- 교육생에게 수익을 보장하거나 최고 운영자 성과를 평균처럼 제시하지 않습니다.
- 고객·상품·리드가 없는 상태에서 가입비·교육비 중심으로 파트너를 모집하지 않습니다.
- 브랜드 사용·운영통제·교육지원·대가 지급 구조는 가맹사업 해당 가능성을 검토합니다.
- 플랫폼·광고·결제·고객 계정을 개인 명의로 방치하지 않습니다.
- 개인정보·영업비밀·고객 파일은 최소 권한으로 운영하고 퇴사·종료 즉시 회수합니다.
- 자동화 오류가 가격·메시지·정산을 전체 단위에 복제하지 않도록 승인·중지·복구를 둡니다.
- 고객·브랜드·데이터·SOP·코드·콘텐츠의 소유와 종료 뒤 사용 범위를 시작 전에 계약합니다.
- OEM·ODM 화장품·건강기능식품은 판매업 신고·품목·표시광고·품질책임을 별도 공식 조사 없이 실행하지 않습니다.
