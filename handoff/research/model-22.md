## 핵심 결론
AI 대량등록 구매대행·오픈마켓 자동화의 공식 정책상 병목은 `총 등록 수가 아니라 활성 상품 비율 × 주문 발생 SKU 비율 × 공급가·재고 동기화 × 배송·반품 정확성 × 천 개 활성 등록당 확정 순이익 × 자동화 중단·복구 능력`입니다. 대량등록은 한 번 만든 규칙을 계속 쓰는 일이 아니라 플랫폼 필수정보·호출 제한·공급처 데이터·이미지 사용조건을 지속적으로 따라가는 운영입니다.

## 성패를 가르는 것
- 총 등록→활성 등록→노출→주문 발생→배송→구매확정→비용 차감 순이익을 단계별로 기록합니다.
- 상품별 공급처·최근 확인일·공급가·재고·옵션코드·배송비·판매중지 기준을 관리합니다.
- SKU마다 최저 허용가와 공급가·환율·배송비 급등 시 판매중지 기준을 둡니다.
- `공급처 원본 → AI 정리 → 규칙 검사 → 사람 승인 → 게시 → 오류 기록` 순서를 사용합니다.
- 주문 수집·공급처 구매·송장·배송·취소·반품·정산을 하나의 상태표로 연결합니다.
- 공급처 선결제액·미정산 매출·취소·반품 예상액·가용자금을 매일 확인합니다.
- API 호출 제한·사양 변경·장애에 대비해 재시도·중복 방지·누락 대조·알림을 둡니다.

## 핵심 인사이트
- 구체적 관찰: 등록 수와 실제 판매 가능한 활성 상품 수는 다릅니다.
- 남들이 놓치는 숨은 변수: 옵션 매핑 오류, 공급가·재고 갱신 지연, 주문 뒤 품절, 이미지 사용기간, API 사양 변경, 선결제 자금입니다.
- 결과가 달라지는 작동 메커니즘: 일정한 마진식으로 넓게 등록한 뒤 주문·클릭·반품 데이터를 이용해 잘 팔리는 공급처·카테고리·가격대를 확대하고 오류 묶음을 중단해야 포트폴리오가 정리됩니다.
- 임계점·시간차·교환비용: 데이터 수급→등록→검색 반영→주문→공급처 결제→배송→구매확정·반품→플랫폼 정산의 시간차가 있습니다.
- 초보자가 먼저 측정하거나 결정할 것: 활성 등록 수, 천 개당 주문·구매확정·순이익, 가격·품절 갱신 지연, 운영자 개입시간입니다.
- 사업을 계속할 조건: 등록 수가 늘 때 사람 개입과 오류가 비례하지 않고 천 개 활성 등록당 순이익이 도구비·CS·세금을 넘는 경우입니다.
- 중단·전환 신호: 활성 등록이 줄거나 주문 뒤 품절·가격변경 취소가 많거나 자동화가 멈춰도 감지하지 못하는 경우입니다.
- 대표 사례·수치·반례: 총 1만 개 등록보다 실제 판매 가능한 3천 개의 재고·가격·옵션이 정확한 편이 운영 가치가 높을 수 있습니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: 쿠팡 API 필수정보·호출 제한, 도매매 다채널 전송과 이미지 사용기간은 공식 사실이며 천 개당 순이익은 편집 판단입니다.

## 시장 현실
- 시장조사를 통해 먼저 확인한 동향: business의 국내 배송대행·중국 구매대행·산업재·계절 상품·다채널 자동화 유형을 유지합니다.
- 반복된 패턴: 공급사 DB를 엑셀·API·전송기로 수급하고 상품명·옵션·카테고리·이미지를 채널별 형식으로 변환합니다.
- 상충되는 의견: 개별 상품을 깊게 최적화하는 운영과 다수 상품을 규칙 기반으로 선별하는 운영은 인력·마진·리스크 구조가 다릅니다.
- 작성자 이해관계 또는 광고 가능성: 대량등록 도구·교육 판매자는 등록 수와 자동화율을 강조하고 취소·반품·제재·운전자금을 낮게 보일 유인이 있습니다.
- 이미 포화된 부분: 동일 공급사 상품을 같은 이미지·문구로 대량 전송하는 생활잡화와 주문 뒤 품절이 잦은 상품입니다.
- 아직 남은 기회: 규격이 세분된 롱테일, 안정된 공급 데이터가 있는 전문상품, 주문 발생 뒤 집중 개선하는 2단계 구조입니다.

## 판매·수익·광고 경로
- 실제 구매자·독자·시청자·광고주: 쿠팡·스마트스토어·G마켓·옥션 등에서 필요한 상품을 구매하는 국내 개인·사업자입니다.
- 발견·유입 경로: 오픈마켓 검색·추천·프로모션·광고입니다.
- 판매 또는 광고가 일어나는 플랫폼: 쿠팡·스마트스토어·11번가·G마켓·옥션 등 판매자센터입니다.
- 결제수단: 각 오픈마켓의 카드·간편결제·계좌 등 결제수단을 따릅니다.
- 수익 지급자: 각 오픈마켓 정산 시스템입니다.
- 플랫폼 수수료·광고비·환불·CS 비용: 공급가·구매·배송대행비·국내외 배송비·판매비용·광고·반품·불량·CS·도구·세금입니다.
- 확정 수익 조건: 주문이 구매확정·취소·반품·분쟁·정산 조정을 통과하고 모든 비용을 차감한 경우입니다.
- 실제 입금 시점과 경로: 플랫폼 정산일과 공급처 결제일이 달라 선결제 자금이 필요할 수 있습니다.
- 반복 가능한 구조인지: 활성 상품·가격·재고·배송·오류를 자동 감시하고 천 개당 순이익이 안정될 때 가능합니다.
- 해외 타깃과 구분할 부분: 국내 구매자·한글 상품정보·원화 정산을 기준으로 하며 해외 공급은 조달 방식으로만 다룹니다.

## 공식 확인 사실
- 쿠팡 OPEN API는 카테고리·물류·브랜드·상품·배송·환불·반품·교환·쿠폰·CS·정산 API를 제공합니다.
- 쿠팡은 2026년 5월 브랜드·GTIN 또는 모델번호·카테고리별 구매옵션 필수정보와 신규 브랜드 API를 도입했습니다.
- 상품 생성 API에 `brandId`와 상품식별자 관련 필드가 추가됐고 기존 필드 일부는 향후 폐기될 수 있습니다.
- 쿠팡은 2026년 3월 17일부터 Open API 기준 호출 빈도를 초당 10회에서 5회로 조정했습니다.
- 쿠팡 OPEN API 장애로 주문 수집 값이 잘못 반환된 과거 공지가 있으며 필요시 주문 재수집을 안내했습니다.
- 도매매는 상품 DB·엑셀 다운로드·엑셀 일괄주문과 배송대행 B2B 서비스를 제공합니다.
- 스피드고전송기는 스마트스토어·쿠팡·11번가·롯데ON 등의 상품 전송·주문·문의·템플릿 관리를 제공합니다.
- 도매매 구매자 매뉴얼은 상품 이미지를 상품 공급기간에 한해서만 무료로 사용할 수 있으며 공급기간 밖 사용은 무단 사용으로 간주될 수 있다고 안내합니다.
- 도매매는 품절확인목록으로 상품의 품절·재판매 상태를 관리하는 기능을 안내합니다.
- 플랫폼별 등록 한도·판매비용·정산일·API 조건은 계정과 정책에 따라 달라질 수 있습니다.

## 계약·계정 화면 확인 필요
- 공급처의 데이터·이미지·상표·번역·상세정보 사용 범위와 공급 종료 뒤 삭제 의무.
- 공급가·재고·옵션·배송비·출고·반품·불량·재배송·증빙 책임.
- 쿠팡·스마트스토어·11번가·ESM PLUS의 현재 등록 한도·판매비용·정산·API 권한.
- 해외 공급 상품의 통관·제품안전·표시·인증·지식재산권 적용 여부.
- 구매·배송대행사의 수수료·환율·검수·보상·분쟁·개인정보 조건.

## 사용자 제공 운영 관찰
- business의 `천 개 활성 등록당 주문·순이익·오류·운영시간`, 주문 발생 상품만 집중 개선하는 2단계 구조를 유지합니다.
- 해외 자동화 사업자의 공개 기능과 성공사례를 국내 운영자의 수익 보장으로 사용하지 않습니다.

## 추정 또는 편집 판단
- 첫 상품군 1개·공급처 1~2곳·채널 1~2개·100개 시험 등록은 초기 검증 기준입니다.
- 천 개 활성 등록당 주문·순이익을 핵심 지표로 보는 것은 포트폴리오 평가를 위한 편집 판단입니다.
- 등록 수가 늘면 매출과 순이익이 비례해 늘어난다는 보장은 없습니다.
- 자동화 시스템을 타인에게 판매하는 것은 자체 운영 실적과 유지보수 조건이 검증된 뒤에만 검토합니다.

## 필요한 제품
쿠팡 OPEN API — 상품·주문·배송·반품·교환·CS·정산 연동. 최신 필수 항목·호출 제한·변경 공지 확인.

도매매·스피드고전송기 — 상품 DB·다채널 전송·주문·품절·문의 관리. 데이터·이미지 사용 범위와 공급기간 확인.

ESM PLUS — G마켓·옥션 상품·주문·정산 관리. 공식 기능과 최신 약관 범위에서 사용.

CapCut — 주문·전환이 검증된 상품군 영상의 자동 자막·다중 플랫폼 편집에 우선 사용. 내부 소재·음악은 Commercial·Dual Use 표시 확인.

## 오늘·3일·7일·30일
- 오늘: 상품군 1개, 공급처 1~2곳, 채널 1개, 최저 허용가와 운전자금 상한을 정합니다.
- 3일: 원본 데이터·옵션코드·필수정보·가격·재고 갱신 흐름과 중단 기준을 만듭니다.
- 7일: 100개 안팎을 시험 등록해 오류·노출·주문·공급처 취소를 기록합니다.
- 30일: 활성 등록·천 개당 주문·순이익·품절·배송·반품·현금 묶임을 비교해 확대 여부를 결정합니다.

## 위험과 주의사항
- 공급처가 허용하지 않은 상품정보와 이미지를 복제·재배포하지 않습니다.
- 공급 종료 뒤 이미지·상세정보를 계속 사용할 수 있다고 가정하지 않습니다.
- AI가 만든 규격·인증·상표·가격·재고를 원자료 확인 없이 게시하지 않습니다.
- 손실 가격 자동화와 실제 공급이 불가능한 상품 노출을 피합니다.
- 주문 누락·중복·송장 지연·품절이 생기면 등록 확대를 멈춥니다.
- 별도 확인이 필요한 상품은 자동 등록 대상에서 제외합니다.

## 총괄이 반드시 확인할 것
- 총 등록 수와 활성·주문 발생·구매확정·순이익을 구분할 것.
- 공급처 가격·재고·배송·반품을 핵심 변수로 다룰 것.
- 공급처 선결제와 오픈마켓 정산 시차를 반영할 것.
- 공급 데이터와 이미지 사용기간을 포함할 것.
- API 사양 변경·호출 제한·장애·재수집을 운영비에 포함할 것.
- 모델 21의 고객사 판매대행과 모델 22의 자체 상품 포트폴리오 운영을 구분할 것.
