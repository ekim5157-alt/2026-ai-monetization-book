## 프로세스 보호 원칙
- business의 모델사례와 사용자 확정 시장 관찰은 검증·교정·복사하지 않는다.
- 공식 사실, 계약·계정 화면 확인 필요, 사용자 제공 운영 관찰, 편집 판단을 분리한다.

## 핵심 결론
AI 업무자동화 구축 대행·컨설팅의 공식 정책상 병목은 `현재 업무의 기준값 측정 × 예외·사람 승인·복구 설계 × 개인정보와 최소 권한 × 사용량 기반 원가 × 구축 뒤 변경·유지보수 계약`입니다. 데모가 작동해도 실제 계정·권한·데이터·보안 검토와 현장 시험을 통과하기 전에는 운영 가능한 납품으로 보기 어렵습니다.

## 성패를 가르는 것
- 월 처리 건수·건당 사람 시간·오류율·오류 1건의 손실을 먼저 측정합니다.
- 정상 흐름보다 누락·중복·오타·긴급·권한 부족·외부 장애를 먼저 목록화합니다.
- 업무를 자동 실행·사람 승인·자동화 금지의 세 등급으로 나눕니다.
- 중복 실행 방지키·입출력 로그·오류 알림·수동 전환·재실행·롤백을 설계합니다.
- 하자·기능 추가·업무 규칙 변경·외부 API 변경·긴급 대응을 계약에서 분리합니다.

## 핵심 인사이트
- 구체적 관찰: 자동화 성공은 데모가 아니라 현장 사용률·예외 복구·절감시간으로 결정됩니다.
- 남들이 놓치는 숨은 변수: 기준 업무, 전면 재검수, 데이터 보존, 접근권한 회수, 승인 대기, 유지보수 부채입니다.
- 결과가 달라지는 작동 메커니즘: 정상·예외·승인·로그·복구를 함께 설계하면 업무가 멈추지 않고 책임 지점을 추적할 수 있습니다.
- 임계점·시간차·교환비용: 문의→진단 계약→설계·PoC→현장 시험→검수·인수→잔금→유지보수의 시간차가 있습니다.
- 초보자가 먼저 측정하거나 결정할 것: 월 처리량·건당 시간·오류비용·자동화 금지 범위·월 지원 상한입니다.
- 사업을 계속할 조건: 고객 절감효과가 유지비보다 크고 공급자의 지원시간 차감 순이익이 남는 경우입니다.
- 중단·전환 신호: 현장 사용률이 낮거나 사람이 결과를 전부 다시 확인하거나 지원시간이 계약금을 넘는 경우입니다.
- 대표 사례·수치·반례: 5분 업무를 하루 2건 줄이는 것보다 30초 업무를 하루 수천 건 줄이는 편이 가치가 클 수 있습니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: API 가격·데이터 보존·학습 제외는 공식 사실이며 세 등급 분류와 20~50건 관찰은 편집 판단입니다.

## 시장 현실
- 시장조사를 통해 먼저 확인한 동향: business의 영업·문의·문서·RAG·데이터·커머스·승인·Agent 업무보조 유형을 유지합니다.
- 반복된 패턴: 챗봇 단독보다 CRM·예약·결제·대시보드·권한·알림과 결합된 운영형 프로젝트가 반복됩니다.
- 상충되는 의견: 맞춤 구축과 표준 패키지는 예외율·데이터 구조·권한 차이에 따라 수익성이 달라집니다.
- 작성자 이해관계 또는 광고 가능성: 자동화 도구·강의·템플릿 판매자는 예외·운영비·유지보수를 낮게 보일 유인이 있습니다.
- 이미 포화된 부분: 일반 FAQ 챗봇, 연결 데모, 표준 템플릿 설치만 제시하는 제안입니다.
- 아직 남은 기회: 한 업종의 예외·승인·감사 로그·복구까지 포함한 운영형 자동화와 작은 PoC입니다.

## 판매·수익·광고 경로
- 실제 구매자·독자·시청자·광고주: 소상공인·쇼핑몰·교육·상담·제조·물류·기관·기업 실무팀입니다.
- 발견·유입 경로: 위시켓·크몽·숨고·네이버 검색·사례 콘텐츠·소개·직접 영업입니다.
- 판매 또는 광고가 일어나는 플랫폼: 유료 진단·직접 계약·프로젝트 플랫폼입니다.
- 결제수단: 플랫폼 대금보호, 계좌이체, 카드·세금계산서 조건을 따릅니다.
- 수익 지급자: 고객 또는 대금보호 플랫폼입니다.
- 플랫폼 수수료·광고비·환불·CS 비용: 진단·설계·개발·시험·교육·API·서버·광고·외주·세금·지원시간입니다.
- 확정 수익 조건: 진단 승인·시험운영·검수·인수와 계약상 잔금 지급을 통과한 금액입니다.
- 실제 입금 시점과 경로: 계약상 착수금·중도금·잔금 일정과 등록 계좌를 따릅니다.
- 반복 가능한 구조인지: 같은 업종·업무의 진단표·예외표·권한표·인수인계 문서를 재사용할 수 있을 때 가능합니다.
- 해외 타깃과 구분할 부분: 한글 업무·원화 견적·원화 계좌 입금을 기본으로 봅니다.

## 공식 확인 사실
- OpenAI API 데이터는 명시적으로 공유에 동의하지 않으면 모델 학습에 사용되지 않습니다.
- 기본 악용 모니터링 로그는 대부분의 API 기능에서 최대 30일 보존될 수 있습니다.
- Responses API는 기본적으로 응답 상태를 최소 30일 저장할 수 있습니다.
- Conversations·ChatKit threads 등 일부 상태형 객체는 삭제할 때까지 보존될 수 있습니다.
- Zero Data Retention과 Modified Abuse Monitoring은 승인 대상이며 모든 엔드포인트에 동일하게 적용되지 않습니다.
- 외부 MCP 서버와 제3자 서비스로 전송한 데이터는 해당 서비스의 보존정책을 따릅니다.
- OpenAI API는 ChatGPT 구독과 별도 사용량 과금입니다.
- 2026년 6월 22일 표준 처리 기준 GPT-5.5는 입력 100만 토큰당 5달러, 캐시 입력 0.50달러, 출력 30달러입니다.
- GPT-5.4는 입력 2.50달러, 캐시 입력 0.25달러, 출력 15달러입니다.
- GPT-5.4 mini는 입력 0.75달러, 캐시 입력 0.075달러, 출력 4.50달러입니다.
- 웹 검색은 1,000회당 10달러이며 Batch API는 입력·출력 비용을 50% 줄이고 최대 24시간 비동기 처리합니다.
- CLOVA Studio는 선택한 모델·도구의 입력·출력 사용량을 기준으로 과금하며 정확한 원화 단가는 이용 시점의 요금 화면을 확인해야 합니다.

## 계약·계정 화면 확인 필요
- 고객 데이터의 수집근거·보존기간·삭제·국외 이전·제3자 처리 조건.
- 조직·프로젝트별 API 키·접근권한·비용 한도·관리자 소유권.
- 외부 CRM·메일·결제·예약·MCP 서비스의 보존·장애·요금 조건.
- 하자·변경·운영지원·긴급대응·종료·소스·문서·교육·인수인계 범위.
- 프로젝트 플랫폼의 대금보호·수수료·검수·분쟁·지급 조건.

## 사용자 제공 운영 관찰
- business의 기준 업무량·오류비용·예외·승인·로그·복구·유지보수 관찰을 유지합니다.
- 프로젝트 관심·지원자 수를 계약 확정이나 공급자 수익으로 사용하지 않습니다.

## 추정 또는 편집 판단
- 실제 작업 20~50건 관찰·세 등급 분류·월 지원시간 상한은 내부 운영 기준입니다.
- API가 사람보다 항상 저렴하거나 정확하다는 보장은 없습니다.
- 데모 성공이 현장 안정성을 뜻하지 않습니다.
- 사용량이 같아도 프롬프트·출력·검색·재시도·파일 처리에 따라 원가가 달라집니다.

## 필요한 제품
OpenAI API — ChatGPT 구독과 별도 사용량 과금. 모델별 입력·캐시·출력·검색 비용과 기능별 데이터 보존 확인.

CLOVA Studio — 한글 업무용 모델·도구 후보. 정확한 원화 사용량 단가와 부가세는 최신 요금 화면 확인.

NAVER WORKS API — 고객 조직의 권한·앱·토큰 범위와 관리자 승인·회수 조건 확인.

Python·자체 서버 — 예외·로그·중복 방지·재처리·비용 한도를 직접 통제할 때 사용. 서버·백업·모니터링 비용 별도.

## 오늘·3일·7일·30일
- 오늘: 반복 업무 1개와 월 처리량·시간·오류·자동·승인·금지 구간을 측정합니다.
- 3일: 정상 흐름·예외·승인·로그·복구를 포함한 진단서와 계약 조건을 만듭니다.
- 7일: 작은 데이터로 시험운영하고 비용·정확도·예외·복구시간을 기록합니다.
- 30일: 절감시간·오류·사용률·지원시간·고객가치·공급자 순이익을 비교합니다.

## 위험과 주의사항
- API 학습 제외를 데이터 무보존으로 설명하지 않습니다.
- 고객의 개인 계정에 종속된 구조보다 고객 조직 소유 프로젝트와 최소 권한을 사용합니다.
- 결제·삭제·발송 등 되돌리기 어려운 동작에는 사람 승인과 중복 방지를 둡니다.
- 무료 체험 비용으로 실제 운영 손익을 계산하지 않습니다.
- 외부 연결 서비스의 데이터 보존·권한·장애 정책을 확인합니다.
- 하자·변경·긴급대응·인수인계 범위를 계약서에 분리합니다.

## 총괄이 반드시 확인할 것
- 기술 연결보다 기준 업무·예외·승인·로그·복구·현장 사용을 중심으로 설명할 것.
- 학습 제외와 악용 모니터링·상태형 데이터 보존을 구분할 것.
- API 가격에는 2026년 6월 22일 기준과 변동 가능성을 표시할 것.
- ChatGPT 구독과 API 비용을 같은 상품으로 설명하지 말 것.
- 사람 승인형 자동화를 실패나 미완성으로 표현하지 말 것.
- 구축비와 유지보수·변경·장애대응을 별도 계약으로 둘 것.
