## 핵심 결론

AI 업무자동화 구축 대행·컨설팅의 공식 정책상 핵심 병목은 **현재 업무의 기준값 측정 × 예외·사람 승인·복구 설계 × 개인정보와 최소 권한 × 사용량 기반 원가 × 구축 뒤 변경·유지보수 계약**입니다.

OpenAI API로 보낸 데이터는 명시적으로 공유에 동의하지 않는 한 모델 학습에 사용되지 않습니다. 그러나 이는 `아무 데이터도 저장되지 않는다`는 뜻이 아닙니다. 기본 악용 모니터링 보존은 여러 API에서 30일이며, Responses API는 기본적으로 응답 상태를 최소 30일 저장할 수 있습니다. Conversations·ChatKit threads 등 일부 상태형 객체는 삭제할 때까지 유지될 수 있습니다. Zero Data Retention은 사전 승인 대상이고 모든 기능에 동일하게 적용되지 않습니다.

API 비용은 ChatGPT 구독과 별도입니다. 2026년 6월 21일 표준 처리 기준 GPT-5.5는 입력 100만 토큰당 5달러, 캐시 입력 0.50달러, 출력 30달러입니다. GPT-5.4는 2.50달러·0.25달러·15달러, GPT-5.4 mini는 0.75달러·0.075달러·4.50달러입니다. 웹 검색은 1,000회당 10달러이며 Batch API는 입력·출력 비용을 50% 줄이는 대신 최대 24시간 비동기 처리입니다. 월 예산 제한은 집행 지연이 있을 수 있으므로 절대 차단 장치로만 의존하면 안 됩니다.

NAVER Cloud Platform의 CLOVA Studio도 선택한 모델·도구의 입력·출력 토큰을 기준으로 종량 과금하며 부가세가 별도입니다. 정확한 요금은 이용 시점의 요금 계산기와 계정 화면을 최종 기준으로 확인해야 합니다.

## 성패를 가르는 것

### 관찰
고객은 `AI로 자동화해 달라`고 말하지만 월 처리량·건당 시간·오류·승인·예외가 정리되지 않은 경우가 많습니다. 정상 데이터로 작동한 데모가 실제 운영에서는 누락·중복·권한 부족·외부 장애 때문에 멈출 수 있습니다.

### 숨은 변수
- 월 처리 건수와 건당 시간
- 오류 한 건의 재작업·보상 비용
- 누락·중복·오타·긴급 처리 비율
- 자동 실행·사람 승인·자동화 금지 구간
- API 입력·출력·검색·파일·재시도 비용
- 수집·전송·저장·백업·삭제 위치
- 관리자·직원·대행사 계정 권한
- 외부 서비스 변경·장애·요금 인상
- 무상 하자와 유상 변경의 경계
- 월 지원시간과 인수인계 범위

### 작동 메커니즘
1. 자동화 전 기준 시간을 측정해야 절감효과와 구축가격을 설명할 수 있습니다.
2. 정상 흐름보다 예외·중복 방지·재실행·상태 로그를 먼저 설계하면 현장 중단을 줄일 수 있습니다.
3. 오류 비용이 낮은 업무는 자동 실행하고 높은 업무는 근거와 함께 사람에게 넘겨야 책임 위험이 낮아집니다.
4. 데이터 흐름과 최소 권한을 문서화하면 고객 자료의 무단 접근과 계정 공유를 줄일 수 있습니다.
5. 하자·기능 추가·고객 업무 변경·외부 API 변경을 계약에서 나누면 무상 지원이 누적되는 것을 막을 수 있습니다.

### 실행 판단
- 실제 작업 20~50건을 관찰해 입력→판단→승인→출력→보관→예외→재작업을 기록합니다.
- 업무를 자동 실행·사람 승인·자동화 금지의 세 등급으로 나눕니다.
- 중복 실행 방지키·오류 로그·알림·수동 전환·재실행·롤백 절차를 만듭니다.
- 고객 소유 계정과 프로젝트별 API 키를 사용하고 최소 권한을 적용합니다.
- 호출당 비용이 아니라 정상 처리·예외·재시도·검색·저장·지원까지 포함한 건당 원가를 계산합니다.
- 계약서에 하자·변경·운영지원·긴급대응·종료·인수인계를 분리합니다.

### 실패·전환 신호
- 기준 시간과 오류를 측정할 수 없다: 개발보다 유료 진단으로 전환합니다.
- 사람이 결과를 전부 다시 확인한다: AI 단계가 추가 업무가 된 것입니다.
- 수동 복구가 원래 업무보다 오래 걸린다: 자동화 범위를 줄입니다.
- 고객이 데이터 위치·접근자·삭제방법을 설명하지 못한다: 연결을 중단합니다.
- 월 지원시간이 계약금을 넘는다: 유지보수 범위와 가격을 다시 정합니다.
- 고객마다 구조가 완전히 다르다: 표준 패키지 범위를 좁힙니다.

## 핵심 현실 인사이트
- 구체적 관찰: 자동화 성공은 데모가 아니라 현장 사용률·예외 복구·절감시간으로 결정됩니다.
- 남들이 놓치는 숨은 변수: 기준 업무·예외율·전면 재검수·데이터 보존·권한 회수·유지보수 부채입니다.
- 결과가 달라지는 작동 메커니즘: 정상·예외·승인·로그·복구를 함께 설계하면 업무가 멈추지 않습니다.
- 임계점·시간차·교환비용: 문의→진단 계약→착수금→설계·구축→시험운영→검수·인수→잔금→유지보수의 시간차가 있습니다.
- 초보자가 먼저 측정하거나 결정할 것: 월 처리량·건당 시간·오류비용·자동화 금지 범위·월 지원 상한입니다.
- 계속할 조건: 고객 절감효과가 유지비보다 크고 공급자의 지원시간 차감 순이익이 남는 경우입니다.
- 중단·전환 신호: 현장 사용률이 낮거나 전면 재검수가 필요하거나 지원시간이 계약금을 넘는 경우입니다.
- 대표 사례·수치·반례: 5분 업무를 하루 2건 자동화하는 것보다 30초 업무를 하루 수천 건 줄이는 편이 가치가 클 수 있습니다. 이는 편집 판단입니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: API 가격·데이터 보존·학습 제외는 공식 확인 사실입니다. 자동·승인·금지 분류는 편집 판단입니다.

## 시장 현실
- 네이버 블로그·카페·브런치·유튜브·커뮤니티에서 먼저 확인한 동향: 주문·상담·견적·보고·문서 처리처럼 반복 업무를 줄이는 소형 자동화 수요가 많습니다.
- 반복된 패턴: 도구 연결 화면과 완전자동화를 강조하고 예외·보안·유지보수·교육을 과소평가합니다.
- 상충되는 의견: 맞춤 구축과 표준 패키지 중 유리한 방식은 고객 업무의 예외와 반복성에 따라 달라집니다.
- 작성자 이해관계 또는 광고 가능성: 자동화 도구·강의·템플릿 판매자는 구현 난도와 운영비를 낮게 보일 유인이 있습니다.
- 이미 포화된 부분: 범용 챗봇·단순 알림 연결·근거 없는 완전자동 응답 데모입니다.
- 아직 남은 기회: 한 업종의 주문·상담·견적·보고·분류를 예외와 승인까지 포함해 표준화한 패키지입니다.

- 반복 가능한 구조인지: 같은 업종·업무의 진단표·예외표·권한표·인수인계 문서를 재사용할 수 있을 때 가능합니다.

## 필요한 제품

OpenAI API — ChatGPT 구독과 별도 사용량 과금. 모델별 입력·캐시 입력·출력·검색 비용 확인. 기본 학습 제외와 보존기간을 엔드포인트별 검토.

CLOVA Studio — 입력·출력 토큰 기반 종량 과금, 부가세 별도. 정확한 모델 요금은 최신 요금 계산기 확인. 한글 업무와 데이터 요구에 맞는지 시험.

NAVER WORKS API — 고객 소유 조직·앱·권한을 사용. 메일·주소록·캘린더·드라이브 접근범위와 관리자 승인·토큰 회수 조건 확인.

Python·자체 서버 — 예외·로그·중복 방지·재처리·비용 한도를 직접 통제할 때 사용. 서버·백업·모니터링 비용 별도.
