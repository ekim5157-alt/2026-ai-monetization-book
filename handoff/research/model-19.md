## 핵심 결론

AI 업무자동화 구축 대행·컨설팅의 공식 정책상 핵심 병목은 **모델 연결이 아니라 데이터 처리 방식, 사용량에 따라 커지는 API·자동화 비용, 최소 권한, 예외 승인, 로그·복구, 구축 후 유지보수 범위**입니다.

OpenAI API에 전송한 데이터는 명시적으로 동의하지 않는 한 모델 학습에 사용되지 않습니다. 그러나 이것이 `데이터가 전혀 저장되지 않는다`는 뜻은 아닙니다. 기본 악용 모니터링 로그는 고객 콘텐츠를 포함할 수 있으며 최대 30일 보관될 수 있습니다. Responses API의 상태 저장, 파일, 대화·벡터 저장소 등은 기능과 설정에 따라 별도 보존기간이 적용됩니다. Zero Data Retention은 승인 대상 고객에게 제공되며 모든 엔드포인트와 기능이 동일하게 지원되는 것도 아닙니다.

비용도 월 구독 하나로 끝나지 않습니다. OpenAI API는 입력·캐시 입력·출력 토큰과 도구 호출별로 과금되며 ChatGPT 구독과 별도입니다. Make 같은 자동화 플랫폼은 모듈 실행마다 크레딧이 차감되고, 재시도·분기·오류처리도 실행량을 늘릴 수 있습니다. 데모 한 건의 비용이 아니라 월간 정상 처리·예외·재시도·모니터링을 포함한 건당 총비용을 계산해야 합니다.

## 성패를 가르는 것

### 1. 자동화 전에 투자 회수 가능한 업무인지 측정할 것

다음 값을 먼저 기록합니다.

- 월 반복 건수와 사람 처리시간
- 오류 한 건의 재작업·보상 비용
- 예외 비율과 승인 필요 비율
- 입력 데이터의 누락·중복·형식 오류율
- 기존 SaaS·API·계정 권한 확보 가능성
- 구축비·월 API비·자동화비·모니터링·유지보수비

절감액보다 구축·유지비가 크거나 예외가 표준 흐름보다 많으면 완전자동화가 아니라 입력 표준화 또는 사람 승인형이 적합합니다.

### 2. `학습 제외`, `로그 보존`, `상태 저장`을 구분할 것

OpenAI API 공식 문서 기준:

- API 데이터는 명시적 옵인 없이는 모델 학습에 사용되지 않습니다.
- 악용 모니터링 로그는 기본적으로 최대 30일 보관될 수 있습니다.
- Chat Completions와 Responses 등은 기본적으로 30일 악용 모니터링이 적용됩니다.
- Responses API에서 상태 저장을 사용하면 응답 데이터가 최소 30일 저장될 수 있습니다.
- Conversations·ChatKit threads 등 일부 상태형 객체는 삭제할 때까지 보관될 수 있습니다.
- Zero Data Retention·Modified Abuse Monitoring은 사전 승인이 필요한 별도 통제이며 기능별 제한이 있습니다.
- 외부 MCP 서버나 연결 서비스로 보낸 데이터는 해당 제3자 서비스의 보존정책을 따릅니다.

따라서 고객에게 `API라서 저장되지 않는다`고 단정하지 말고 실제 엔드포인트·store 설정·파일 만료·외부 연동을 데이터 흐름도로 기록해야 합니다.

### 3. API 비용을 모델 가격만으로 계산하지 않을 것

2026년 6월 21일 OpenAI 공식 표준 처리 가격 예시:

- GPT-5.4 mini: 입력 100만 토큰당 0.75달러, 캐시 입력 0.075달러, 출력 4.50달러
- GPT-5.4: 입력 2.50달러, 캐시 입력 0.25달러, 출력 15달러
- GPT-5.5: 입력 5달러, 캐시 입력 0.50달러, 출력 30달러
- 웹 검색: 1,000회 호출당 10달러
- Batch API: 입력·출력 비용 50% 절감, 최대 24시간 비동기 처리

실제 비용에는 긴 컨텍스트, 재시도, 실패 요청, 도구 호출, 이미지·음성, 검색, 저장소, 자동화 플랫폼 실행량과 클라우드가 포함됩니다. 예산 제한은 집행 지연이 있을 수 있으므로 사후 청구 초과 가능성도 계약에 반영해야 합니다.

### 4. 자동화 플랫폼의 실행 단위를 이해할 것

Make 공식 가격 페이지 기준 Free 플랜은 월 1,000크레딧, 최소 실행 간격 15분이며, 모듈 동작 하나가 일반적으로 1크레딧으로 계산됩니다. 유료 Make 플랜은 5,000크레딧 기준 월 9달러가 표시되지만 결제 주기·지역·상품 개편에 따라 달라질 수 있습니다.

한 업무가 1회 실행되는 것과 한 크레딧은 같은 개념이 아닙니다. 검색·분기·행 추가·API 호출·재시도마다 여러 크레딧이 소모될 수 있습니다. 무료 한도로 PoC가 작동해도 실제 운영량에서 비용이 급증할 수 있습니다.

### 5. 최소 권한·승인·복구를 납품물로 포함할 것

운영 자동화에는 다음이 필요합니다.

1. 고객 소유 계정과 프로젝트별 API 키
2. 필요한 데이터와 기능만 허용하는 최소 권한
3. 결제·발송·삭제·외부 공개 전 사람 승인
4. 입력·출력·오류·승인·재시도 로그
5. 중복 실행 방지 키와 실패 재처리 기준
6. 비용 한도와 이상 사용 알림
7. 장애 시 수동 처리와 롤백 절차
8. 담당자 변경을 고려한 문서·교육·계정 회수

데모가 성공하는 것과 실제 운영에서 예외를 안전하게 처리하는 것은 다른 성과입니다.

## 공식 확인 사실

1. OpenAI API 데이터는 명시적 옵인 없이는 모델 학습에 사용되지 않습니다.
2. 기본 악용 모니터링 로그는 고객 콘텐츠를 포함할 수 있고 최대 30일 보관될 수 있습니다.
3. 기능별 애플리케이션 상태 보존기간은 서로 다르며 일부 객체는 삭제 전까지 유지됩니다.
4. Zero Data Retention은 승인된 고객에게 제공되며 지원하지 않는 엔드포인트·기능이 있습니다.
5. OpenAI API 이용료는 ChatGPT Plus·Business·Enterprise 구독에 포함되지 않고 별도 과금됩니다.
6. OpenAI API는 모델별 입력·캐시 입력·출력과 도구 사용량에 따라 과금됩니다.
7. Make는 모듈 동작을 크레딧으로 계산하며 Free 플랜은 월 1,000크레딧과 15분 최소 실행 간격을 안내합니다.
8. 개인정보를 국외 서비스에 전송하거나 외부 수탁자가 처리하면 개인정보보호법상 처리근거·고지·위탁·국외이전·안전조치를 검토해야 합니다.

## 추정 또는 편집 판단

다음은 공식 보장 기준이 아닙니다.

- 반복 업무라면 모두 자동화하는 것이 이익이라는 주장
- API 사용이 사람보다 항상 저렴하다는 주장
- 데모 성공이 실제 운영 안정성을 뜻한다는 주장
- AI 정확도가 높으면 사람 승인이 불필요하다는 주장
- 고객의 SaaS 계정에 연결 권한이 있으면 모든 데이터를 처리해도 된다는 주장
- 월 유지보수 시간이 구축 전에 정확히 고정된다는 주장

예외율·사람 검수시간·도입률·복구시간을 핵심 지표로 보는 것은 사업성 판단입니다.

## 오늘·3일·7일·30일

- 오늘: 한 업무의 건수·시간·오류·예외·승인과 현재 비용을 측정합니다.
- 3일: 데이터 흐름, 권한, 사람 승인, 실패·복구 경로를 그립니다.
- 7일: 고객 소유 계정의 작은 유료 파일럿으로 비용·정확도·예외를 측정합니다.
- 30일: 절감액·오류비용·API비·유지시간·현장 사용률로 확대 또는 중단을 결정합니다.

## 위험과 주의사항

1. 고객 비밀번호와 개인 API 키를 대행사 개인 계정에 장기 보관하지 않습니다.
2. API 학습 제외를 무보존으로 설명하지 않습니다.
3. 결제·삭제·발송 등 되돌리기 어려운 동작에는 사람 승인과 중복 방지 장치를 둡니다.
4. 무료 플랜 비용으로 실제 운영 손익을 계산하지 않습니다.
5. 외부 MCP·SaaS·클라우드로 전달되는 데이터의 별도 보존정책을 확인합니다.
6. 장애 대응·모델 변경·SaaS API 변경·추가 수정의 유상 범위를 계약에 명시합니다.
7. 선금·파일럿 승인·잔금·월 유지비 입금을 분리합니다.

## 총괄이 반드시 확인할 것

- 기술 연결보다 업무 진단·예외 처리·권한·복구·현장 도입을 중심으로 설명할 것.
- OpenAI API의 학습 제외와 최대 30일 악용 모니터링 로그를 구분할 것.
- API 가격은 공식 확인일을 붙이고 변동 가능성을 명시할 것.
- 한 업무 실행과 자동화 플랫폼의 크레딧 소비를 같은 것으로 쓰지 말 것.
- 사람 승인형 자동화를 실패나 미완성으로 표현하지 말 것.
- 구축과 유지보수·변경요청·장애대응을 계약상 별도 범위로 둘 것.
