# HANDOFF

## 기본 정보
- 모델 번호: 14
- 모델명: 템플릿·프롬프트팩·워크북 판매
- 담당 역할: 사업성 분석
- 작성일: 2026-06-21
- 공식 확인일: 조사 담당 확인 전

## 핵심 결론
핵심은 파일 수가 아니라 **반복 업무 빈도 × 첫 사용 성공 × 수정 가능성 × 재사용률 × 지원·개정비 차감 후 순이익**입니다. 예쁜 화면과 프롬프트 개수는 관심을 만들 수 있지만, 복사·권한·입력·수식·변수 변경에서 막히면 사용되지 않습니다.

## 성패를 가르는 것
1. 매주·매월 반복되는 업무를 고릅니다.
2. 다운로드보다 설명 없이 첫 결과가 나오는지를 봅니다.
3. 핵심 구조, 사용자가 바꿀 변수, 건드리면 안 되는 수식·연결을 구분합니다.
4. 프롬프트팩은 사용 시점·입력자료·출력형식·실패 수정법·검수 절차를 포함합니다.
5. 워크북은 질문 수가 아니라 최종 선택안·실행 순서·중단 기준을 남깁니다.
6. 업무 원리와 자주 바뀌는 화면·모델 안내를 분리합니다.
7. 단품 수를 늘리기보다 고객 질문→견적→제안→납품처럼 흐름을 연결합니다.

## 수익 전 임계점과 작업량
- 사용 전후 절약시간 측정
- 초보자가 설명 없이 첫 결과 완성
- 자기 상황에 맞게 핵심 변수 수정
- 7일·30일 뒤 재사용
- 환불·문의 뒤 건당 순이익 유지
- 도구 변경 뒤 개정시간 감당

최소 기능 제작→초보자 테스트→첫 사용 실패 수정→예시 데이터 보완→반복 문의를 FAQ·영상으로 전환→재사용 확인 순으로 진행합니다.

## 관심이 돈이 되지 않는 이유
사용영상 조회수는 도구 관심, 무료 다운로드는 수집 행동일 수 있습니다. 유료 파일 전달은 첫 사용 성공이 아니며 후기 수는 재사용을 보장하지 않습니다. 환불 전 매출은 확정 수익이 아닙니다.

## 숨은 병목 5개
1. 첫 사용 실패
2. 자기 업무에 맞게 수정하지 못함
3. 도구·모델 변경으로 빠르게 낡음
4. 무료 대체재와 차이가 없음
5. 설치·권한·수식 문의로 지원비가 누적됨

## 측정 지표
| 지표 | 판단 |

| 사용 빈도 | 반복 가치 |
| 첫 사용 완료율 | 실제 사용성 |
| 첫 결과까지 시간 | 온보딩 난이도 |
| 7일·30일 재사용률 | 반복 가치 |
| 구매자당 문의시간 | 숨은 원가 |
| 환불률 | 기대 일치 |
| 개정당 수정시간 | 유지비 |
| 비용 차감 순수익 | 실제 사업성 |

## 첫 수익과 실제 입금
`노출→방문→결제→파일 전달·복제→첫 사용→환불·분쟁 기간→정산 확정→계좌 입금`
결제는 제품 성공이 아닙니다.

## 실패·중단·전환
프롬프트 수만 많거나, 제작자만 수정할 수 있거나, 도구 변경 뒤 깨지거나, 낮은 가격 뒤 문의가 폭증하면 실패합니다. 첫 사용률이 낮으면 기능보다 온보딩을 고칩니다. 재사용이 낮으면 반복 업무인지 재검증합니다. 맞춤 요청이 많으면 설치·세팅·상담 서비스로 전환합니다.

## 시장 현실과 판매 경로
- 구매자: 취업 준비자, 프리랜서, 소상공인, 콘텐츠 제작자, 업무를 표준화하려는 팀
- 발견: 네이버 블로그·카페, 브런치스토리, 유튜브 사용영상, SNS, 뉴스레터
- 판매: 크몽 등 디지털 상품 판매처, 자체 결제페이지, Notion 템플릿 마켓
- 결제: 신용카드, 네이버페이, 카카오페이, 토스페이, 계좌이체
- 지급: 판매 플랫폼·템플릿 마켓·결제대행사
- 비용: 판매·결제 수수료, 환불, 광고비, 지원·개정시간

## 공식 확인 사실
사업성 분석 단계에서는 가격, 무료 한도, 재판매 조건, 수수료, 지급일을 확정하지 않습니다.

## 현실 기준 교정
원화 가격·간편결제·계좌 입금을 먼저 확인합니다. 해외 판매는 환율·환전·세금·지원시간을 포함하며 영어권의 Notion·prompt 구매 습관을 그대로 적용하지 않습니다.
