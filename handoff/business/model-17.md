## 핵심 결론
AI 업무자동화 구축 대행은 챗봇이나 자동화 도구를 설치하는 일이 아니다. 고객의 현재 업무를 수치로 정리하고, 자동 실행할 일과 사람이 승인할 일, 실패했을 때 복구할 일을 계약하는 사업이다. 최근 프로젝트 시장에는 B2B 영업 자동화, 고객 문의, 문서·보고서, 데이터 수집·분석, 커머스·재고 연동, 사내 지식 검색, 예약·상담, 권한·대시보드, AI Agent UI·운영이 함께 나타난다.

성패는 `기준 업무량과 오류비용이 측정되는가`, `예외가 목록화되는가`, `권한·로그·복구가 설계되는가`, `변경·유지보수 대가를 받을 수 있는가`에서 갈린다. 구축비만 보고 계약하면 고객 업무가 바뀔 때마다 무료 수정이 쌓여 손실이 난다.

## 성패를 가르는 것
- 구체적 관찰: 위시켓은 2026년 6월 현재 AI Agent, 데이터 분석, 내부 시스템, 업무자동화·RPA, 스크래핑·API, AX 전환, 유지보수 프로젝트를 별도 시장으로 노출한다.
- 남들이 놓치는 숨은 변수: 자동화 성공률보다 예외 1건의 비용, 승인권자, 입력 데이터 품질, 시스템 계정 소유권, 장애 때 수동 복귀 시간이다.
- 작동 메커니즘: 기준 업무를 먼저 측정하면 자동화 전후 시간을 비교할 수 있고, 예외·승인·로그를 정의하면 운영자가 결과를 신뢰할 수 있다.
- 임계점·시간차·교환비용: 데모가 작동해도 실제 계정·권한·데이터·보안 검토와 현장 테스트를 통과하기 전에는 납품이 아니다.
- 초보자가 먼저 측정할 것: 월 처리 건수, 건당 사람 시간, 오류율, 예외 유형, 승인 대기시간, 오류 1건의 손실, 유지보수 요청 수.
- 사업을 계속할 조건: 고객 한 곳에서 만든 모듈과 체크리스트를 같은 업종의 다른 고객에게 60~80% 재사용할 수 있어야 한다.
- 중단·전환 신호: 고객이 기준 프로세스를 설명하지 못함, 데이터 접근 승인이 없음, 시스템 API가 없음, 예외가 계속 새로 생김, 책임자가 결과를 승인하지 않음, 유지보수 계약을 거부함.

## 발견·계약·지급 경로
`프로젝트 공고·소개·직접 영업 → 현행 업무 진단 → 범위·예외·데이터·권한 확정 → 견적·계약 → 착수금 또는 대금보호 → PoC → 현장 검수 → 인수인계 → 잔금 → 변경·유지보수`

- 실제 돈을 내는 주체: 기업·기관의 사업부, 운영팀, 마케팅팀, 관리부서, 대표자.
- 수익 지급자: 위시켓 거래는 플랫폼의 대금보호 절차를 거친 고객 예치금, 직접 계약은 고객사 재무부서.
- 확정 수익: 산출물과 검수 기준을 통과하고 고객이 인수 승인해야 한다. 운영 데이터가 없으면 PoC 성공과 본 구축 매출을 분리한다.
- 실제 입금: 착수금·중도금·잔금 일정 또는 플랫폼 지급 일정에 따른다. 세금계산서·검수 문서·보안 승인 때문에 지연될 수 있다.

## 작업량·기간·순이익
- 진단: 인터뷰 2~5회, 업무 흐름도, 처리량·오류·예외 표본 수집.
- PoC: 대표 업무 1개, 제한된 데이터, 승인 화면, 실패 로그로 작동 여부 확인.
- 본 구축: 실제 계정·권한·보안·모니터링·백업·복구·교육을 추가.
- 안정화: 2~4주 이상 실제 운영에서 예외와 오류를 기록하고 수정.
- 유지보수: API·모델·업무 규칙 변경, 사용량 증가, 장애 대응을 별도 계약.

`프로젝트 순이익 = 수금액 - 개발·기획·디자인 인건비 - 모델·서버·도구비 - 외주비 - 출장·회의 - 무상수정·장애대응 - 플랫폼 수수료 - 세금 충당액`

## 초보자가 잘못 읽기 쉬운 숫자
- 자동화율: 사람이 검수·재처리한 시간을 빼지 않으면 과장된다.
- 처리 건수: 쉬운 건만 자동화하고 어려운 예외를 사람이 맡으면 전체 절감이 작을 수 있다.
- 정확도: 오류 1건의 비용이 크면 95%가 충분하지 않을 수 있다.
- 구축 기간: 고객의 데이터·권한·승인 대기일을 제외한 개발일만 보고 계약하면 손실 난다.
- 문의·지원자 수: 계약·착수금·잔금이 아니다.

## 반복 가능한 구조
- 업종별 기준 업무 진단표를 만든다.
- 입력·판단·출력·예외·승인·로그를 모듈로 분리한다.
- 고객별 변경은 설정값과 템플릿에서 처리하고 핵심 코드는 공통으로 유지한다.
- 사람에게 넘기는 조건과 수동 복구 절차를 제품 일부로 납품한다.
- 월 유지보수에는 사용량, 장애 대응시간, 포함 변경시간, 초과 단가를 적는다.

## 위험과 주의사항
- 개인정보·민감정보를 외부 AI에 전송하기 전 처리 근거와 계약을 확인한다.
- 고객 계정 비밀번호를 개인 메신저로 받지 않고 권한 위임·비밀관리 절차를 둔다.
- 삭제·송금·발송·계약 변경처럼 되돌리기 어려운 작업은 사람 승인을 둔다.
- 모델 출력은 확률적이므로 로그·재시도·검수·복구가 필요하다.
- 외부 API 가격·정책·한도 변경은 하자가 아니라 변경 업무가 될 수 있음을 계약에 쓴다.
