## 핵심 결론

당근은 유료광고, 비즈니스 프로필, 개인샵, 피드, 커뮤니티, 후기·평가·이웃·팔로우가 연결되는 신뢰 기반 사업 플랫폼입니다.

모델 19는 다음 세 축을 함께 다룹니다.

1. 유료광고 직접 운영
2. 온라인 도매상과 연결한 상품·농산물 박스 판매
3. 커뮤니티 코칭을 강의·전자책·멤버십·상담·SaaS로 확장

`광고 승인 순이익 + 커머스 확정 순이익 + 교육·구독 순이익 = 당근 기반 전체 순이익`

세 축은 프로필·후기·팔로우를 공통 신뢰 자산으로 사용하지만 비용과 수익 장부는 분리합니다.

## 성패를 가르는 것

### 1. 유료광고는 클릭 부업이 아니라 캠페인 운영입니다
- 관찰: 광고비를 직접 집행해 피드 노출·문의·상담을 만들 수 있습니다.
- 숨은 변수: 광고 상품, 타기팅, 소재 심사, 클릭당 비용, 문의당 비용, 무효 리드, 상담 계약률, 설치·개통, 취소·철회·환수입니다.
- 작동 메커니즘: 클릭이 문의·유효 리드·계약·설치·승인까지 지나야 수익이 확정됩니다.
- 실행 판단: `광고비 → 노출 → 클릭 → 문의 → 유효 리드 → 계약 → 설치·개통 → 승인 → 환수 반영 순이익`을 기록합니다.
- 실패·전환 신호: 승인 수익이 광고비·소재비·상담비보다 작으면 예산을 늘리지 않습니다.

정수기, 인터넷·통신 결합상품처럼 상담·설치·개통이 필요한 상품은 대표 사례로 볼 수 있습니다. 광고 약속과 상담 조건이 일치해야 합니다.

### 2. 프로필 신뢰와 광고 성과를 분리해 봅니다
- 관찰: 후기·평가·팔로우는 신뢰를 높이지만 광고 전환을 자동으로 만들지는 않습니다.
- 숨은 변수: 첫 이미지, 대상·조건 문구, 문의 마찰, 후기의 실제 근거, 리드 개인정보 동의입니다.
- 작동 메커니즘: 대상·조건·진행 과정·제외 조건이 명확하면 낮은 의도의 문의가 줄어듭니다.
- 실행 판단: 소재별 클릭·문의·유효 리드·승인 수익과 프로필 방문·팔로우를 따로 기록합니다.
- 실패·전환 신호: 팔로우는 늘지만 유효 문의가 늘지 않으면 소재와 프로필 역할을 다시 나눕니다.

### 3. 농산물 박스 판매는 독립된 커머스 축입니다
- 관찰: 온라인 도매상과 공급 계약을 맺고 산지 상품을 박스 단위로 등록하면 판매자는 재고·포장·배송을 직접 보유하지 않고도 운영할 수 있습니다.
- 숨은 변수: 도매상 재고, 선별·포장 기준, 품질 편차, 출고일, 파손·부패, 반품, 판매대금 지급, 후기, 반복 구매입니다.
- 작동 메커니즘: 판매자는 상품 선택·산지와 생산자 이야기·개인샵 운영·문의·후기·재구매를 담당하고, 도매상은 재고·포장·배송을 담당합니다.
- 실행 판단: `도매상 계약 → 상품·공급 조건 수급 → 실제 산지·생산자 정보로 박스 등록 → 주문 → 도매상 포장·배송 → 수령·환불 → 판매대금 지급 → 반복 구매`를 관리합니다.
- 실패·전환 신호: 품질 편차·파손·배송 지연·환불 차감이 반복되거나 도매상 정보와 게시물이 다르면 판매를 중단합니다.

쿠팡보다 가격이 높더라도 후기와 신뢰가 쌓이면 반복 구매가 생길 수 있습니다. 산지·고향·생산자 이야기는 실제 공급 구조와 일치해야 하며 허위로 만들지 않습니다.

### 4. 개인샵·택배 판매는 직거래와 손익 구조가 다릅니다
- 관찰: 직거래에서 쌓인 후기와 팔로우는 개인샵과 택배 주문의 불안을 낮출 수 있습니다.
- 숨은 변수: 샵 이동률, 주문 전환, 포장·택배비, 결제수수료, 취소·반품, CS, 반복 구매, 정산 시차입니다.
- 작동 메커니즘: 콘텐츠와 커뮤니티가 프로필 신뢰를 만들고, 배송 경험이 다시 후기·팔로우를 늘리면 판매 범위를 넓힐 수 있습니다.
- 실행 판단: 직거래·택배·개인샵 주문의 매출·비용·반품·재구매를 각각 계산합니다.
- 실패·전환 신호: 팔로우는 늘지만 샵 이동이 없거나 택배·반품 비용이 마진을 없애면 상품과 가격을 바꿉니다.

### 5. 농산물 세금·증빙은 상품 상태에 따라 확인합니다
- 관찰: 농산물이라고 모두 같은 세금 처리가 적용되는 것은 아닙니다.
- 숨은 변수: 미가공 여부, 가공·혼합·소분·포장 방식, 판매자와 공급자의 증빙, 배송비, 환불·파손 차감입니다.
- 작동 메커니즘: 상품 상태와 계약 구조가 달라지면 면세 적용 범위와 장부 처리도 달라질 수 있습니다.
- 실행 판단: 상품별로 세금 적용, 공급 증빙, 판매대금 지급일, 환불·파손 차감 기준을 공식 자료와 계약서로 확인합니다.
- 실패·전환 신호: 공급자가 증빙·정산·반품 기준을 제시하지 못하면 상품 등록을 보류합니다.

### 6. 커뮤니티 코칭은 교육 플랫폼으로 확장합니다
- 관찰: 실행형 모임은 질문·실패 패턴·성과 사례를 모으는 진입 방식입니다.
- 숨은 변수: 참여율, 재참여, 과제 제출, 결과물 완성, 반복 질문, 피드백시간, 후기, 후속 상품 전환입니다.
- 작동 메커니즘: 질문·과제·결과물·실패 패턴을 표준화하면 강의·전자책·멤버십·상담·SaaS로 전환할 수 있습니다.
- 실행 판단: `커뮤니티 모집 → 목표·과제·피드백 → 질문·결과물·후기 → 커리큘럼 표준화 → 강의·전자책·멤버십·상담·SaaS`로 설계합니다.
- 실패·전환 신호: 피드백시간이 참가비를 넘거나 후속 상품 구매가 없으면 운영 범위를 줄입니다.

### 7. SaaS는 커뮤니티에서 반복 확인된 문제만 해결합니다
- 관찰: 커뮤니티 뒤에 도구를 붙이는 것만으로 유료 제품이 되지 않습니다.
- 숨은 변수: 반복 질문 빈도, 기존 해결시간, 체험 뒤 재사용, 사용자당 AI·서버비, 해지, 자료 사용 동의입니다.
- 작동 메커니즘: 실제 참가자가 반복해서 겪는 문제를 선택하면 제품 선택 실패를 줄일 수 있습니다.
- 실행 판단: 반복 질문을 빈도·시간·오류비용으로 순위화하고 한 문제만 시험합니다.
- 실패·전환 신호: 주간 재사용이 없거나 사용자당 비용이 결제액보다 크면 개발을 확대하지 않습니다.

### 8. 원소스멀티유즈는 세 축의 수요를 연결합니다
- 관찰: 광고 소재, 상품 시연, 커뮤니티 과제·성과를 하나의 원본에서 채널별로 재가공할 수 있습니다.
- 숨은 변수: 경제적 이해관계 표시, 사례 사용 동의, 채널별 링크 목적, 제작시간입니다.
- 작동 메커니즘: 유튜브·인스타그램·틱톡·X·페이스북에 맞춰 재가공하면 당근 밖에서도 수요를 만들 수 있습니다.
- 실행 판단: 광고 문의, 개인샵 주문, 커뮤니티 신청의 전환을 각각 측정합니다.
- 실패·전환 신호: 채널 수만 늘고 승인 수익·구매확정·유료 신청이 늘지 않으면 채널을 줄입니다.

## 핵심 현실 인사이트
- 구체적 관찰: 광고 클릭·상품 주문·커뮤니티 참가와 확정 수익은 서로 다른 사건입니다.
- 남들이 놓치는 숨은 변수: 승인·환수, 농산물 품질 편차, 도매상 정산, 개인샵 반복 구매, 과제·후속 상품 전환입니다.
- 결과가 달라지는 작동 메커니즘: 거래·참여 후기와 팔로우가 광고·커머스·교육의 신뢰 마찰을 낮춥니다.
- 임계점·시간차·교환비용: 광고는 설치·승인, 상품은 배송·반품, 교육은 제공·환불·구독 조건 뒤 수익이 확정됩니다.
- 초보자가 먼저 측정하거나 결정할 것: 첫 사업축 1개, 유입 태그, 비용 장부, 최종 수익 조건입니다.
- 계속할 조건: 한 축의 순이익이 확인되고 공통 신뢰 자산이 다른 축의 전환도 보강하는 경우입니다.
- 중단·전환 신호: 노출·문의·팔로우는 늘지만 승인 수익·구매확정·유료 전환이 늘지 않는 경우입니다.
- 대표 사례·수치·반례: 광고 클릭 수천 회보다 설치·개통 완료 계약이 중요하고, 농산물 주문 수보다 반품 뒤 반복 구매가 중요합니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: 세 사업축 연결은 편집 판단이며 광고·개인샵·도매상·결제 조건은 공식 확인이 필요합니다.

## 시장 현실
- 주요 구매자: 생활 서비스 상담 수요, 상품 구매자, 스터디·취미·실습 참가자, 소상공인·프리랜서·강사
- 주요 유입: 당근 피드·커뮤니티·검색·유료광고·선생님 프로필·비즈니스 프로필·후기·평가·이웃·팔로우
- 판매·교육 연결: 개인샵, 허용된 외부 판매페이지, 클래스101·탈잉·크몽, 브런치스토리·전자책 유통, 자체 판매페이지
- 주요 비용: 광고비·상담비·환수, 상품 원가·포장·배송·반품·CS, 커뮤니티 운영·강의 제작·플랫폼 수수료·SaaS 비용
- 반복 조건: 캠페인 순이익, 상품 반복 구매, 교육 후속 전환이 각각 반복되어야 합니다.

## 판매·수익·광고 경로
- 유료광고: `광고비 → 문의 → 유효 리드 → 상담 → 계약 → 설치·개통 → 승인·환수 → 입금`
- 농산물 박스: `도매상 계약 → 개인샵 등록 → 주문 → 도매상 포장·배송 → 반품·파손 차감 → 판매대금 지급`
- 교육: `커뮤니티 → 과제·피드백 → 강의·전자책·멤버십·상담·SaaS 결제 → 환불·정산 → 입금`
- 수익 지급자: 상담형 제휴 운영사, 상품 판매처·PG사, 강의·전자책·구독 판매처
- 대상 범위 교정: 한글 콘텐츠·원화 결제·원화 정산과 실제 허용 기능을 기준으로 합니다.

## 공식 확인 사실
광고 상품·과금·심사·타기팅, 상담형 제휴 허용, 개인샵·결제·택배·정산, 커뮤니티·외부 링크·참가비·멤버십 조건은 다음 단계가 공식 문서로 확인합니다.

## 현실 기준 교정
- 판매자 형태별 적용 가능 여부: 개인과 개인사업자의 광고·개인샵·교육·제휴·통신판매업·세금 조건을 확인합니다.
- 결제·지급·정산 조건: 광고 승인 수익, 상품 판매대금, 강의·전자책·구독 정산을 분리합니다.
- 원화 가격·세금·수수료: 광고비·환수·판매·배송·교육 플랫폼 수수료·부가가치세·종합소득세를 포함합니다.
- 상품 적용 확인: 농산물의 면세 범위, 가공·혼합·소분·포장 예외, 증빙, 파손·환불 차감을 확인합니다.
- 적용 확인이 필요한 항목: 유료광고·상담형 제휴·개인샵·택배·외부 링크·커뮤니티 참가비·SaaS 연결입니다.

## 추정 또는 편집 판단
- 세 사업축을 연결하되 장부를 분리하는 것은 편집 판단입니다.
- 농산물 박스 판매는 대표 사례이며 다른 상품군에도 공급·배송·반품 역할 분담을 적용해 검토할 수 있습니다.
