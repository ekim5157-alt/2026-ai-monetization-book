## 핵심 결론
링크를 많이 뿌리는 방식보다 구매 직전의 비교·사용 상황·대체품 정보를 만들고, 허용 매체와 광고 표시를 지키며, 클릭→주문→취소·반품 차감→승인→입금을 상품군별로 기록해야 한다.

## 성패를 가르는 것
### 구체적 관찰
- 상품명만 나열하기보다 누구에게 어떤 상황에서 필요한지를 먼저 설명한다.

- 직접 촬영·사용 화면이 없는 대량 복제 글보다 원본 경험이 있는 콘텐츠가 신뢰를 만든다.
- 경제적 이해관계 문구를 눈에 보이게 표시하고 허용 매체를 계정에 등록한다.

### 남들이 놓치는 숨은 변수
- 클릭 귀속 기간과 다른 기기·앱 이동에서 추적이 끊기는 조건
- 상품군별 수수료와 제외·변경 항목
- 취소·반품이 승인 수익을 줄이는 시간차
- 낮은 상품가에서 콘텐츠 제작비가 남는지
- 가격·재고·상품 페이지 변경을 관리하는 시간

### 작동 메커니즘
발견 지표가 커져도 다음 행동과 승인 수익이 따라오지 않으면 사업이 아니다. 콘텐츠·상품·서비스의 약속이 맞을 때 `노출 → 시청·클릭 → 문의·주문 → 취소·환불 통과 → 승인 → 입금`이 이어진다. 각 단계가 끊기는 이유를 따로 기록한다.

### 초보자가 먼저 측정하거나 결정할 것
- 한 콘텐츠를 만드는 데 실제로 들어간 조사·촬영·편집·검수·응대시간
- 콘텐츠별 노출·시청·클릭·문의·주문·승인·입금
- 도구·광고·샘플·교통·외주·환불·CS를 차감한 이익
- 30일 동안 반복 가능한 주제와 중단할 주제

### 조사 결과 때문에 기존 가정에서 수정한 부분
기존 원고의 ‘좋은 콘텐츠를 많이 만들면 수익이 난다’는 범위를 축소했다. 최근 시장에서는 같은 노출이어도 구매 경로, 승인 조건, 취소·환불, 제작시간, 플랫폼 밖 자산의 유무에 따라 결과가 크게 갈린다. 따라서 조회수·구독자·다운로드를 대표 성과로 두지 않고 승인 수익과 실제 입금, 반복 생산비를 중심으로 다시 구성했다.

## 판매·수익·광고 경로
- 실제로 돈을 내는 사람: 위 유형별 구매자·광고주·수강생·예약자·기업 고객
- 발견·유입 경로: 검색, 추천 피드, 관련 콘텐츠, 댓글·공유, 이메일, 프로필, 판매페이지
- 판매 또는 광고가 일어나는 곳: 해당 플랫폼의 광고·쇼핑 기능, 제휴 판매처, 예약·상담 페이지, 자체 결제페이지
- 결제수단: 판매처가 제공하는 신용카드·간편결제·계좌이체·자동결제
- 수익 지급자: 플랫폼, 광고주, 제휴 운영사, 판매처, PG사, 고객
- 역할 분담: 콘텐츠 운영자는 상품 선택·사용·비교·링크 표시를 맡는다. 쿠팡은 상품 페이지·결제·배송·환불을 처리하고, 쿠팡파트너스는 추적된 승인 구매에 대해 수수료를 지급한다. 구매자는 쿠팡의 결제수단을 사용한다.
- 확정 수익 조건: 광고 검수·월 마감, 주문의 취소·반품 기간 통과, 예약의 실제 이용 완료, 서비스 납품·검수, 유료 구독의 환불·결제 실패 반영
- 실제 입금 시점과 경로: 지급자별 대시보드에서 승인된 뒤 약정 지급일에 등록 계좌로 입금. 구체적인 날짜·최소 지급액·세금은 다음 단계에서 확인한다.
- 반복 가능한 구조: 한 건의 바이럴이 아니라 같은 고객 문제를 여러 소재로 만들고 콘텐츠별 승인 이익이 양수일 때만 반복한다.

## 결과가 바뀌는 임계점
- 클릭이 적으면 콘텐츠가 구매 단계와 멀거나 링크가 찾기 어렵다.
- 클릭은 많고 주문이 없으면 상품·가격·판매 페이지 적합성이 낮다.
- 주문은 많고 승인 수익이 적으면 취소·반품·제외 상품·수수료를 확인한다.
- 상품 업데이트와 링크 교체시간이 승인 이익보다 크면 상품군을 줄이거나 오래 판매되는 품목으로 바꾼다.
