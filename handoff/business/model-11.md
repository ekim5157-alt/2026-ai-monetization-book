## 핵심 결론

쿠팡파트너스는 링크를 많이 뿌리는 사업이 아닙니다. 성패는 **구매 직전 질문을 해결하는 콘텐츠 × 정확한 상품 매칭 × 취소·반품을 통과한 확정 수익 × 광고비·촬영비·콘텐츠 제작시간 차감 후 순이익**에서 갈립니다.

`구매 의도 × 상품 일치율 × 링크 클릭 × 주문 확정률 × 수수료 × 제작·광고비 차감 순이익`

조회수와 클릭은 관심일 뿐입니다. 주문이 발생해도 취소·반품·무효 처리 뒤 수수료가 확정되고 정산을 거쳐 계좌에 입금되어야 실제 수익입니다.

## 성패를 가르는 것

### 구매 직전 질문을 잡아야 합니다
- 관찰: 제품명만 나열한 글보다 크기·호환·단점·대체품을 다룬 콘텐츠가 구매에 가깝습니다.
- 숨은 변수: 구매 시점, 가격대, 배송 필요성, 규격·모델명, 반품 가능성입니다.
- 작동 메커니즘: 시청자나 독자가 마지막으로 확인하려는 질문을 해결하면 링크 클릭과 확정 구매가 늘어납니다.
- 실행 판단: 상품마다 `누가 왜 사는가`, `무엇과 비교하는가`, `사면 안 되는 경우`를 기록합니다.
- 실패·전환 신호: 클릭은 많은데 주문 확정이 적으면 상품 적합성·가격·과장 표현을 점검합니다.

### 상품 매칭 오류가 신뢰와 수익을 동시에 깎습니다
- 관찰: 오프라인에서 본 제품과 링크 상품의 규격·구성·판매자가 다를 수 있습니다.
- 숨은 변수: 모델명, 옵션, 묶음 수량, 가격 변동, 재고, 판매자입니다.
- 작동 메커니즘: 유사 상품을 동일 상품처럼 소개하면 반품과 신뢰 하락이 생깁니다.
- 실행 판단: 모델명·규격·옵션을 확인하고 동일 제품이 아니면 차이를 명시합니다.
- 실패·전환 신호: 반품·문의가 반복되면 해당 상품군을 중단합니다.

### 채널마다 링크 마찰이 다릅니다
- 관찰: 네이버 블로그, YouTube, Instagram, Threads, 인포크링크 허브의 링크 접근 방식이 다릅니다.
- 숨은 변수: 클릭 단계, 모바일 속도, 프로필 링크 구조, 상품명 검색 난이도입니다.
- 작동 메커니즘: 콘텐츠의 상품명과 도착 페이지가 정확히 일치할수록 이탈이 줄어듭니다.
- 실행 판단: 게시물마다 고유 번호를 붙이고 링크 허브 첫 화면에서 같은 번호로 찾게 합니다.
- 실패·전환 신호: 링크 클릭 뒤 상품 페이지 도달이 낮으면 허브 구조를 단순화합니다.

### 저가 상품은 이동·촬영비를 회수하지 못할 수 있습니다
- 관찰: 생활용품은 구매 빈도가 높아도 건당 수익이 작을 수 있습니다.
- 숨은 변수: 촬영·이동시간, 제품 구매비, 반품률, 콘텐츠 수명, 묶음 구매 가능성입니다.
- 작동 메커니즘: 건당 수익이 작으면 같은 촬영에서 여러 상품과 후속 콘텐츠를 만들어야 손익이 맞습니다.
- 실행 판단: 상품군별 건당 확정 수익과 콘텐츠당 총시간을 기록합니다.
- 실패·전환 신호: 30일 순수익이 이동·촬영시간을 회수하지 못하면 고단가·묶음 상품군으로 전환합니다.

## 핵심 현실 인사이트
- 구체적 관찰: 주문 발생과 확정 수익은 다릅니다.
- 남들이 놓치는 숨은 변수: 옵션 일치, 취소·반품, 링크 마찰, 촬영·이동시간입니다.
- 결과가 달라지는 작동 메커니즘: 구매 직전 질문과 정확한 상품을 연결하면 확정률이 올라갑니다.
- 임계점·시간차·교환비용: `콘텐츠 노출 → 링크 클릭 → 주문 → 취소·반품 기간 → 수수료 확정 → 정산 → 계좌 입금`
- 초보자가 먼저 측정하거나 결정할 것: 상품군, 확정률, 건당 수익, 콘텐츠당 시간입니다.
- 계속할 조건: 같은 상품군에서 확정 구매와 순이익이 반복되는 경우입니다.
- 중단·전환 신호: 클릭은 늘지만 확정 수익과 입금이 늘지 않는 경우입니다.
- 대표 사례·수치·반례: 조회 10만의 저가 잡화 글보다 조회 1만의 고관여 비교 글이 더 많은 순이익을 만들 수 있습니다.
- 반복 확인된 구조인지 단일 사례의 편집 판단인지: 클릭보다 확정 구매와 순이익을 우선하는 것은 사업성 판단이며 수수료·확정 기간은 공식 확인이 필요합니다.

## 첫 수익과 실제 입금의 시간 차이

`네이버·YouTube·Instagram·Threads 노출 → 쿠팡 링크 클릭 → 주문 → 취소·반품 기간 → 수수료 확정 → 정산 → 등록 계좌 입금`

## 시장 현실

- 구매자: 생활용품·가전·패션·취미·업무도구 비교 소비자
- 유입: 네이버 블로그·카페, YouTube, Instagram, Threads, 틱톡
- 판매 플랫폼: 쿠팡
- 결제수단: 쿠팡이 제공하는 신용카드·간편결제·계좌 관련 수단
- 수익 지급자: 쿠팡파트너스
- 비용: 촬영·제품·교통·광고·링크 관리·콘텐츠 제작시간
- 반복 가능성: 상품군별 확정률과 순이익이 반복되어야 합니다.

## 판매·수익·광고 경로

- 콘텐츠 경로: 비교·사용법·가격·실패 조건 콘텐츠 → 쿠팡 상품 페이지
- 광고 경로: 네이버 검색광고·Meta 광고는 클릭비용이 확정 수익보다 낮을 때만 시험합니다.
- 확정 수익 조건: 주문이 취소·반품·무효 처리 없이 확정된 경우
- 실제 입금: 쿠팡파트너스 정산주기를 거쳐 등록 계좌로 입금
- 세금: 개인과 개인사업자의 소득 처리·지급서류·부가가치세 적용 여부를 확인합니다.

## 공식 확인 사실

사업성 분석 단계에서는 가입 자격, 수수료율, 쿠키·기여 기준, 취소·반품 확정 기간, 최소 지급액, 지급일을 확정하지 않습니다. 다음 단계가 공식 문서로 확인해야 합니다.

## 현실 기준 교정

- 쿠팡 상품과 콘텐츠의 모델명·옵션·가격을 직접 확인합니다.
- 주문을 수익으로 기록하지 않고 확정 수수료와 입금을 기록합니다.
- 광고비·촬영비·제품비·제작시간을 순이익 계산에 포함합니다.
- 개인과 개인사업자의 가입·세금·정산 조건을 구분합니다.

## 위험과 주의사항

제휴 광고임을 표시하고, 가격·재고·옵션 변동을 고지합니다. 구매하지 않은 제품을 실제 사용한 것처럼 표현하지 않고, 허위 후기·자기구매·무효 클릭 유도·과장 효과 표현을 피합니다.

## 총괄이 반드시 확인할 것

1. 클릭·주문·확정 수익·입금을 구분했는가.
2. 상품 매칭과 취소·반품을 핵심 변수로 다뤘는가.
3. 플랫폼·결제·정산·세금·광고 경로가 구체적인가.
4. 확인 전 수수료율·지급일을 단정하지 않았는가.
5. 문서 마지막에 조사 체크리스트가 있는가.
