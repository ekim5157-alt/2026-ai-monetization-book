# Review handoff

전체 작업의 네 번째 단계입니다.

검증 전에 같은 모델의 business, research, draft 파일을 모두 읽습니다.

## 모델사례 보호 원칙

- 기존에 삽입된 모델사례와 앞으로 추가되는 모든 모델사례에 동일한 보호 원칙을 적용합니다.
- business 파일의 `모델사례` 단락은 사용자 확정 원문입니다.
- 리뷰 과정에서는 모델사례의 사실관계와 문장을 검증하거나 수정하지 않습니다.
- 모델사례를 축약·재작성·재배치·삭제하지 않습니다.
- 모델사례에 대한 수정 지적이나 교정안을 review 파일에 작성하지 않습니다.
- 사용자가 지시하지 않은 Git 연결·생성·저장·우회 방법을 연구하거나 실행하지 않습니다.

검토 항목:
- 실제 병목과 수익 경로가 설명됐는가
- 긴 자료가 일반론으로 평탄화되지 않았는가
- 조건·임계점·반례·숫자가 남아 있는가
- 조회수·클릭·예약·주문과 확정 수익·실제 입금을 구분했는가
- 공식 출처 없는 가격·자격·수수료를 단정했는가
- 미국식 단가·전환율·구독 구조를 그대로 옮겼는가
- 구매자, 유입, 결제, 정산, 비용과 입금 경로가 연결되는가

치명적 오류:
- 누가 돈을 내는지 알 수 없음
- 실제 결제 또는 수익 지급 경로가 없음
- 미국 성공사례만 있고 실행 근거가 없음
- 긴 원자료의 인과관계가 사라지고 일반론만 남음

결과는 `handoff/review/model-XX.md`에 저장합니다. 완료 후 `status/model-status.csv`의 `검증완료`를 갱신합니다.
