# 모델 18. 소형 AI SaaS·GPT·웹앱

- 상태: 초보자 실행 원고 v1
- 선행 문서: `handoff/business/model-18.md`, `handoff/research/model-18.md`
- 문장 형식 기준: `final/model-01.md`
- 공식 확인일: 2026-06-21
- 작성일: 2026-06-22

## 1. 한 줄 결론

소형 AI SaaS·GPT·웹앱은 **좁고 반복되는 문제 × 10분 안의 첫 가치 × 사용자당 AI·서버·결제·지원 원가 × 잔존·해지·결제 실패 × 데이터 삭제와 장애 대응**이 맞아야 수익모델이 됩니다.

## 2. 이 모델은 무엇인가

견적·보고·분류·상담 준비처럼 반복되는 일을 웹앱으로 해결하고 월 구독이나 건별 결제를 받는 모델입니다. 기능 수와 가입자보다 첫 결과, 7일·30일 재사용, 사용자당 마진이 중요합니다.

## 3. 성패를 가르는 것

`누가 어떤 자료를 넣고 어떤 결과를 몇 번 다시 쓰는가`를 한 문장으로 정합니다. 최소 입력으로 10분 안에 첫 결과를 만들게 합니다.

기능별 평균·최대 API, 검색, 파일, 저장, 재시도와 지원비를 측정합니다. 무료 한도와 고비용 기능에 상한을 둡니다.

신규·첫 결과·활성·유료·잔존·해지·결제 실패를 분리합니다. 결제 상태와 서비스 권한을 대조하고 데이터 보유기간, 탈퇴, 삭제, 백업과 관리자 접근 절차를 준비합니다.

## 4. 누가 돈을 지급하는가

사용자가 PG사나 판매 플랫폼을 통해 결제하고 환불·부분취소·결제 실패·해지 조정 뒤 정산된 금액이 입금됩니다. 월 순이익은 결제액에서 AI·서버·DB·PG·환불·지원·광고·개발비를 뺀 금액입니다.

## 5. 실행 경로

반복 문제 하나와 구조화된 결과 하나를 정합니다. 예시 데이터로 최소 화면을 만들고 가입→첫 입력→결과→저장→재사용 흐름을 측정합니다.

첫 사용자 약 20명에게 제공해 기능별 비용, 오류, 문의, 7일·30일 재사용을 기록합니다. 재사용이 확인되기 전에는 기능과 광고를 늘리지 않습니다.

## 6. 외국인·해외 확장

먼저 한글 사용자와 원화 결제를 검증합니다. 해외 확장은 통화, 세금, 결제수단, 데이터 이전, 고객지원 언어와 모델 비용을 별도로 계산합니다.

## 7. 주요 타겟

소상공인, 1인 사업자, 프리랜서, 교육·상담·쇼핑몰·기관 실무자처럼 같은 결과를 반복해서 만드는 사용자가 주요 타겟입니다.

## 8. 부가 타겟

기존 구축 고객의 공통 기능을 표준 제품으로 바꾸거나 초기 세팅 서비스를 함께 판매할 개발자·컨설턴트에게 적합합니다.

## 9. 수익구조

월 구독, 건별 크레딧, 조직 요금제, 초기 세팅, 맞춤 연동으로 나눕니다. 사용 빈도가 낮으면 월 구독보다 건별 결제가 적합할 수 있습니다.

## 10. 첫 수익 예상

기존 고객 문제가 분명하면 1~8주 안에 초기 세팅이나 첫 유료 사용자를 만들 수 있습니다. 안정적인 사업성은 30일 잔존과 사용자당 마진이 확인된 뒤 판단합니다.

## 11. 실제 입금 예상

`콘텐츠 → 가입 → 첫 결과 → 결제 → 반복 사용 → 해지·환불·결제 실패 → PG 정산 → 계좌 입금`의 순서를 따릅니다.

## 12. 필요한 제품과 공식 링크

OpenAI API — ChatGPT 구독과 별도 사용량 과금입니다. 모델·검색·파일·보존 조건을 확인합니다. https://platform.openai.com/

토스페이먼츠 — 일반·자동결제, 웹훅, 환불과 정산을 확인합니다. https://www.tosspayments.com/

Supabase·Vercel — 인증·DB·저장·배포에 사용할 수 있습니다. 무료 한도와 초과요금, 백업을 확인합니다. https://supabase.com/ · https://vercel.com/

CapCut — 제품 시연 영상과 자동 자막을 만드는 1순위 도구입니다. https://www.capcut.com/

## 13. 오늘 실행

반복 문제 하나, 첫 결과, 10분 안의 흐름, 사용자당 비용 상한, 결제 방식을 정합니다.

## 14. 3일 실행

예시 데이터와 최소 화면을 만들고 기능별 비용표, 오류 코드와 삭제 절차 초안을 만듭니다.

## 15. 7일 실행

첫 사용자에게 제공해 첫 결과 완료율, 재시도, 문의, 기능별 최대 비용을 측정합니다.

## 16. 30일 실행

7일·30일 재사용, 유료 전환, 잔존, 해지, 결제 실패, 사용자당 순이익을 비교합니다. 신규 결제와 해지가 같은 속도로 늘면 광고를 중단합니다.

## 17. 자동화

온보딩, 사용량 집계, 결제 상태 대조, 실패 알림, FAQ, 탈퇴와 삭제 요청 흐름을 자동화할 수 있습니다.

## 18. 사람이 직접 할 일

사람은 문제 선택, 첫 가치, 가격, 지원 범위, 데이터 보유와 삭제, 관리자 권한, 장애·환불 기준을 결정합니다.

## 19. 실패 원인

범용 챗봇을 만드는 것, 가입자를 가치로 착각하는 것, 무료 사용량과 최대 비용을 측정하지 않는 것, 결제 상태와 서비스 권한이 불일치하는 것이 대표적인 실패입니다.

## 20. 중단·전환 기준

첫 결과 뒤 재사용이 없으면 문제를 바꿉니다. 사용자가 늘수록 손실이 커지면 모델·길이·무료 한도·가격을 바꿉니다. 지원시간이 결제액을 넘으면 고객군과 기능을 좁힙니다.

## 21. 위험과 주의사항

API 데이터의 학습 제외와 무보존을 혼동하지 않습니다. 기능별 보존기간, 파일 삭제, 백업과 관리자 접근을 확인합니다. 결제 인증과 승인 결과를 서버에서 검증하고 해지 절차를 명확히 합니다.
