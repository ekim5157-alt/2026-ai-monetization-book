# 모델 22. AI 대량등록 구매대행·오픈마켓 자동화

- 상태: 초보자 실행 원고 v1

- 공식 확인일: 2026-06-21

## 1. 한 줄 결론

AI 대량등록 구매대행·오픈마켓 자동화는 **활성 상품 비율 × 주문 발생 SKU 비율 × 공급가·재고 동기화 × 배송·반품 정확성 × 천 개 등록당 확정 순이익 × 자동화 복구 능력**이 맞아야 수익모델이 됩니다.

## 2. 이 모델은 무엇인가

재고를 대량 선매입하기보다 공급처 상품 데이터를 여러 오픈마켓에 등록하고 주문 뒤 공급처에서 구매·배송하는 주문연동형 모델입니다. 총 등록 수보다 실제 판매 가능한 활성 상품, 주문 발생 비율, 구매확정 순이익과 오류율을 봅니다.

## 3. 성패를 가르는 것

`총 등록 → 활성 등록 → 노출 → 주문 발생 → 배송 → 구매확정 → 비용 차감 순이익`을 단계별로 기록합니다. 상품별 공급처, 최근 확인일, 공급가, 재고, 옵션코드, 배송비와 판매중지 기준을 관리합니다.

SKU마다 최저 허용가와 공급가 급등 시 판매중지 기준을 둡니다. 경쟁가를 따라 자동으로 가격을 내리기 전에 수수료·광고·배송·반품비를 모두 넣습니다.

상품 데이터는 `공급처 원본 → AI 정리 → 규칙 검사 → 사람 승인 → 게시 → 오류 기록` 순서로 처리합니다. 주문 수집, 공급처 구매, 송장, 배송, 취소·반품, 환불, 정산을 하나의 상태표로 연결합니다.

공급처 선결제액, 미정산 매출, 취소·반품 예상액과 가용자금을 매일 확인합니다. 주문 증가보다 운전자금 부족이 먼저 올 수 있습니다.

## 4. 누가 돈을 지급하는가

오픈마켓 구매자가 결제하고 플랫폼이 구매확정과 반품 조정 뒤 판매자에게 정산합니다. 공급처에는 판매자가 먼저 결제할 수 있으므로 매출과 현금흐름은 다릅니다.

## 5. 실행 경로

상품군 하나, 공급처 1~2곳, 채널 하나로 시작합니다. 100개 안팎을 시험 등록해 오류·노출·주문·공급처 취소와 배송을 확인합니다.

자동화가 안정된 뒤에만 등록 수를 늘립니다. 최소 3개월의 활성 등록, 천 개당 주문·순이익, 오류율과 중단시간을 증빙하기 전 자동화 시스템 판매를 주요 수익으로 삼지 않습니다.

## 6. 외국인·해외 확장

기본 모델은 국내 공급처와 국내 오픈마켓입니다. 해외 구매대행은 통관, 환율, 배송기간, 반품, 상표·인증과 세금을 별도 사업으로 계산합니다.

## 7. 주요 타겟

공급처 데이터와 주문 후 구매·배송 흐름을 직접 관리할 수 있고 운전자금과 CS를 감당할 수 있는 판매자에게 적합합니다.

## 8. 부가 타겟

검증된 운영 흐름을 구축·세팅·교육·유지보수 서비스로 판매하려는 운영자에게 적합합니다. 실제 운영 증빙이 먼저입니다.

## 9. 수익구조

상품별 확정 마진과 자동화 시스템의 구축·교육·유지보수로 나눌 수 있습니다. 등록·수정·배송·반품·CS·도구·서버·현금 묶임 비용을 차감합니다.

## 10. 첫 수익 예상

첫 주문은 1~4주 안에 가능할 수 있으나 확정수익은 배송·구매확정·반품 뒤에 결정됩니다. 대량 확대는 30일 이상 운영지표를 확인한 뒤 진행합니다.

## 11. 실제 입금 예상

`상품등록 → 주문 → 공급처 선결제 → 배송 → 구매확정·반품 → 오픈마켓 정산 → 계좌 입금`의 순서를 따릅니다.

도매매·스피드고전송기 — 공급상품, 다채널 전송, 주문, 품절, 가격변동과 문의를 관리합니다. 데이터와 이미지 사용 범위를 확인합니다. https://domeme.com/ · https://speedgo.domeggook.com/

## 13. 오늘 실행

상품군 하나, 공급처 1~2곳, 채널 하나, 최저 허용가와 운전자금 상한을 정합니다.

## 14. 3일 실행

원본 데이터, 옵션코드, 공급가·재고 갱신 흐름, 품절·가격급등 중지 기준과 주문 상태표를 만듭니다.

## 15. 7일 실행

100개 안팎을 시험 등록해 등록 오류, 활성 비율, 노출, 주문, 공급처 취소, 송장과 배송을 기록합니다.

## 16. 30일 실행

활성 등록, 천 개당 주문·순이익, 품절, 배송 지연, 반품, 자동화 중단시간과 현금 묶임을 비교합니다. 비율이 악화되면 등록을 줄입니다.

## 17. 자동화

상품 정리, 규칙 검사, 전송, 가격·재고 갱신, 주문 수집, 송장·상태 기록을 자동화할 수 있습니다. 손실 가격, 품절, 실패 호출에는 자동 중지와 알림을 둡니다.

## 18. 사람이 직접 할 일

사람은 공급처 사용권, 규격·인증·상표·이미지 권리, 최저가격, 주문 결제, 배송·반품, 환불과 현금흐름을 검수합니다.

## 19. 실패 원인

총 등록 수만 늘리는 것, 공급가·재고 갱신이 늦는 것, 손실 가격을 자동 적용하는 것, 주문 이후 흐름을 자동화 범위에서 빼는 것, 운전자금을 계산하지 않는 것이 대표적인 실패입니다.

## 20. 중단·전환 기준

공급처 취소·품절·가격 역전이 반복되면 해당 상품군을 제거합니다. 주문 누락·송장 지연·반품지 혼선이 생기면 등록 확대를 멈춥니다. 천 개당 순이익이 하락하면 자동화와 상품 데이터를 수정합니다.

## 21. 위험과 주의사항

공급처가 허용한 범위 밖에서 상품정보와 이미지를 사용하지 않습니다. AI가 만든 규격·인증·가격·재고를 원자료 확인 없이 게시하지 않습니다. API 사양과 호출 제한은 변경될 수 있으므로 실패 재처리와 중단 규칙을 둡니다.
