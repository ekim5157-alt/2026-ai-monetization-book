# Draft handoff

전체 작업의 세 번째 단계입니다.

초보자 실행 원고를 쓰기 전에 반드시 같은 모델의 다음 파일을 읽습니다.

- `handoff/business/model-XX.md`
- `handoff/research/model-XX.md`

사업성 분석과 공식자료 조사에 많은 시간이 투입되어 있으므로 원자료를 삭제하거나 다시 조사하지 않습니다. 검증 정보는 보존하고, 독자에게 불필요한 조사용 표지와 반복 문구만 draft 단계에서 정리합니다.

## 공식자료 문구 정리 규칙

공식자료 조사 문서에 다음과 같은 검토 행이 있더라도 독자용 원고에 그대로 복사하지 않습니다.

- `원화 가격·세금·환율·수수료:`는 `가격·세금·환율·수수료:`로 바꿉니다.
- `언어 지원·지역 제한:`으로 시작하는 행은 제거합니다.
- `플랫폼·소비 습관 차이:`로 시작하는 행은 제거합니다.
- `적용 확인이 필요한 항목:`으로 시작하는 행은 제거합니다.

다음 정보는 삭제하지 않습니다.

- 가격의 환율·카드 결제비용·세금 영향
- 개인 프리랜서와 사업자의 계약·증빙·원천징수·계좌 조건 차이
- 착수금·중도금·잔금·유지보수비·실제 입금 구분
- 개인정보·보안·접근권한 중 실제 실행과 비용에 영향을 주는 내용

삭제 대상은 조사 단계의 반복적인 분류 행이지, 그 안에서 확인된 중요한 사실과 비용·계약 조건이 아닙니다. 필요한 사실은 본문 흐름에 자연스럽게 통합합니다.

## 원고 작성 원칙

- business와 research를 그대로 복사하지 않습니다.
- 긴 자료를 평균적인 요약문으로 평탄화하지 않습니다.
- 반복과 주변 설명은 덜어내되 결과를 가르는 조건, 인과관계, 임계점, 반례와 숫자는 남깁니다.
- 핵심 인사이트는 `관찰 → 숨은 변수 → 작동 메커니즘 → 실행 판단 → 실패·전환 신호`로 씁니다.
- 조회수·클릭·문의와 결제·확정 수익·실제 입금을 구분합니다.
- 초보자가 오늘 바로 실행할 수 있도록 제품명, 공식 링크, 오늘·3일·7일·30일 행동을 구체적으로 작성합니다.

결과는 `handoff/draft/model-XX.md`에 저장합니다. 완료 후 `status/model-status.csv`의 해당 모델 `초안완료` 상태를 갱신합니다.
