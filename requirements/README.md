# Model requirements

이 폴더는 사용자가 직접 정의한 사업 구조와 반드시 보존해야 할 사례를 저장합니다.

## 문서 우선순위

모델별 작업에서는 다음 순서를 적용합니다.

1. `AGENTS.md`
2. `requirements/model-XX.md`
3. `rules/공통_편집규칙.md`
4. 단계별 프롬프트
5. 기존 handoff 원고

`requirements/model-XX.md`가 존재하면 기존 business·research·draft 원고보다 우선합니다. 기존 원고가 요구사항과 충돌하거나 항목을 빠뜨렸다면 기존 원고를 유지하지 않고 같은 경로에서 다시 생성합니다.

## 요구사항 변경 시 무효화

모델 요구사항이 추가되거나 변경되면 해당 모델의 기존 단계 완료 표시는 자동으로 유효하지 않은 것으로 봅니다.

재생성 순서:

1. 상태를 `재생성대기`로 바꾸고 분석·조사·초안·검증·최종 완료 표시를 비웁니다.
2. requirements를 읽고 business를 같은 경로에서 다시 작성합니다.
3. 모델 요구사항 검사를 통과한 뒤에만 분석완료로 바꿉니다.
4. research는 requirements와 새 business를 모두 읽고 다시 작성합니다.
5. research 요구사항 검사를 통과한 뒤에만 조사완료로 바꿉니다.
6. draft·review·final도 requirements를 필수 입력으로 읽습니다.

## 사실과 사용자 제공 사업 구조

사용자가 제공한 운영 사례가 공식 문서에서 바로 확인되지 않는다는 이유로 삭제하지 않습니다.

각 항목은 다음 중 하나로 분류합니다.

- 공식 확인 사실
- 계약서·계정 화면·사업자 인터뷰로 확인할 사항
- 사용자 제공 운영 관찰
- 편집 판단

공식 근거가 부족하면 `확인 필요`로 표시하고 조사 체크리스트에 남깁니다. 항목 자체를 일반적인 플랫폼 설명으로 대체하지 않습니다.

## 완료 조건

business와 research는 각 요구사항 문서의 `필수 포함 검사`를 모두 충족해야 합니다. 핵심 사례를 한 단어만 언급하는 것으로 통과하지 않으며, 역할 분담·수익 흐름·숨은 변수·검증 과제가 연결되어야 합니다.
